# 공고관리·상세모달 점검 메모 (2026-05-29)

## 공고관리 선택 작업

- 공고관리 상단 조작은 `삭제 -> 입력일 -> 종목 -> 재전처리 -> 2차분류 -> 검색` 순서다.
- 종목은 목록상자 하나로 관리한다. 별도 종목 칩 영역은 만들지 않는다.
- 종목 목록상자의 그룹은 `종목_매핑` 마스터의 `일반_기타_전문` 값을 기준으로 한다.
- `삭제`, `재전처리`, `2차분류`는 체크된 row만 대상으로 한다.
- 재전처리 후 필터에서 빠지는 row가 생겨도, 이미 체크한 row는 선택 대상에 남긴다. 작업 기준은 현재 화면 필터가 아니라 공고관리 전체 row의 선택 key다.

## 상세모달 우측 패널

탭 순서:

```text
추가 -> 종목 -> 기본 -> 기타 -> 첨부파일 -> 적격심사기준
```

공간 확보를 위해 제거한 영역:

- 좌측 공고번호/공사명 헤더
- 우측 `채워진 컬럼 / 전체 컬럼` 요약 바

유지한 영역:

- 상단 선택 공고 row 스트립
- 좌측 공고문/첨부 탭
- 좌측 공고문 검색
- 우측 탭별 입력 섹션
- 하단 `저장 / 닫기`

## 좌측 공고문 검색

- 검색 입력 후 Enter: 다음 결과로 이동
- Shift+Enter: 이전 결과로 이동
- Escape 또는 X: 검색 해제
- Ctrl+F/Cmd+F: 모달 안 검색창으로 포커스
- 카운터는 `현재/전체` 형식이다.
- 현재 구현은 HTML/텍스트 공고문 대상이다. PDF 내부 검색은 브라우저 PDF 뷰어 기능을 사용한다.

## 실제 흐름 검수

브라우저에서 다음 흐름을 실행했다.

```text
입찰공고 원본 API 호출
-> 곡괭이전나라시
-> 입찰공고 전처리
-> 문서곡괭이질 시작 후 중단
-> 공고관리 423건 생성
-> 종목=건축 선택
-> 전체 선택
-> 재전처리
-> 2차분류
-> 2차분류 완료 54건 이동 확인
```

확인한 보완점:

- 문서곡괭이질 전체 실행은 오래 걸려 브라우저가 멈춘 것처럼 보일 수 있다. 곡괭이전나라시처럼 chunk 진행률 표시가 필요하다.
- 재전처리 후 종목이 바뀌면 현재 필터 결과가 바뀐다. 선택 대상은 필터가 아니라 전체 공고관리 row 기준으로 유지하도록 수정했다.

## 예시/질문이 더 필요한 목록

1. `추가` 탭의 `협정 상세` 표시 조건
   - 예: `참여형태=단독`이면 협정 상세 섹션을 숨길지, 빈 섹션으로 둘지 확정 필요.

2. `종목` 탭의 `종목세부JSON` 직접 편집 범위
   - 예: 사람이 JSON을 수정하면 다음 재전처리에서 어느 필드까지 다시 계산할지 확정 필요.

3. `동일실적` 세부 입력 UI
   - 예: `참가자격기준규모`, `평가기준`, `만점실적`, `최소인정규모`, `실적평가기간`, `평가비율`을 어떤 반복 블록 구조로 둘지 예시 필요.

4. `첨부파일` 탭의 파일 미리보기 우선순위
   - 예: PDF, HWP, XLSX가 같이 있을 때 어떤 파일을 먼저 열지, 탭 이름을 어떻게 줄일지 예시 필요.

5. `적격심사기준` 탭의 트리 표시 수준
   - 예: 처음부터 전체 펼침인지, 1단계만 펼침인지, 가격/실적 항목을 강조할지 예시 필요.

6. 공고문 검색의 PDF 처리
   - 예: PDF는 앱 검색 불가 안내만 할지, PDF 텍스트 변환 후 별도 검색을 붙일지 결정 필요.

7. 상세모달에서 저장 후 다음 공고 이동
   - 예: `저장 / 닫기`만 둘지, 기존처럼 `다음 빈칸으로` 버튼을 다시 둘지 결정 필요.

## 2026-05-30 공고관리 종목 완료/카운트 기준

공고관리의 종목 선택은 포함검색이 아니라 정확한 토큰 일치다.

- `종목` 컬럼을 `/`, `,`, 괄호 기준으로 나눈다.
- 나뉜 토큰과 선택한 종목이 정확히 같을 때만 해당 종목 공고로 본다.
- 예: `건축` 선택 시 `실내건축`은 불러오지 않는다.
- 예: `조경` 선택 시 `조경식재`는 불러오지 않는다.

완료 표시는 선택한 종목만 완료로 표시한다.

- 2차분류 완료로 이동할 때 `2차분류_종목`에 현재 선택 종목을 저장한다.
- 종목 목록에서 완료 표시는 `2차분류_종목` 값만 본다.
- 예: `건축/조경` 공고를 `건축`으로 2차분류 완료하면 `건축 완료`만 표시한다.
- 같은 공고번호는 공고관리 미완료 대상에서 제외되므로 `조경` 카운트에서도 빠진다.

2차분류 완료 탭에도 선택 체크박스를 둔다.

- 삭제 버튼은 체크한 완료 row만 삭제한다.
- 삭제한 완료 row를 공고관리로 되돌릴지 완전 삭제로 볼지는 추가 결정이 필요하다.

상세모달 공고문 검색은 원문 HTML을 텍스트로 다시 그리지 않는다.

- A3 HTML이 있으면 원본 HTML 구조를 유지한다.
- 검색어와 일치하는 텍스트 노드 부분만 `mark`로 감싼다.
- 표, 문단, 줄바꿈, 원래 스타일은 가능한 한 보존한다.

첨부파일/적격심사기준 API 연결 현황:

- 첨부파일 탭은 `/api/bid-files?gongsanum=...`를 통해 A2 `bidFile_get.php`를 호출한다.
- 적격심사기준 탭은 `/api/qualification?적격기준세부=...&건설=...`를 통해 A4 `qualification.php`를 호출한다.
- 둘 다 우측 탭을 열 때 lazy load 한다. 값이 없으면 안내문을 표시한다.
