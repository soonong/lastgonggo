# 상세모달 우측패널 필드 레이아웃 (2026-05-31)

## 기준

공고관리 상세모달의 우측 패널은 사람이 값을 입력하고 확인하는 영역이다.
탭 순서는 다음을 유지한다.

```text
추가 -> 종목 -> 기본 -> 기타 -> 첨부파일 -> 적격심사기준
```

## 공통 레이아웃

- 필드는 섹션 단위로 나눈다.
- 섹션 헤더에는 접기/펼치기 버튼, 섹션명, 빈칸 개수를 표시한다.
- 필드는 한 줄 구조를 기본으로 한다.

```text
항목명 | 입력칸
```

- 긴 값은 `textarea`를 사용한다.
- 일반 값은 한 줄 `input`을 사용한다.
- 컬럼입력목록이 있는 컬럼은 `select`를 사용한다.
- 체크 성격의 컬럼은 `checkbox`를 사용한다.

## 추가 탭

추가 탭은 보조 입력값과 사람이 검토해야 하는 조건값을 다룬다.

섹션:

1. 공사현장
2. 협정 상세
3. 특수조건

`특수실적`, `특수실적_공통`, `공고확인`, `참가자격`은 일단 `textarea`로 둔다.
향후 사용자가 목록 선택형으로 바꾸라고 지정하면 컬럼입력목록 기반 select로 전환한다.

## 입력 목록

입력 목록은 공고수집의 `공고분류컬럼 설정`에서 관리하는 `선택목록` 값을 사용한다.
화면 표시명은 `컬럼입력목록`이지만, 기존 데이터 호환을 위해 저장 컬럼명은 `선택목록`을 유지한다.

## 공고문 하이라이트

우측 입력칸에 포커스가 가면 좌측 공고문에서 관련 키워드를 하이라이트한다.

검색 키워드 우선순위:

1. `신문서곡괭이` 설정의 같은 항목 검색키워드 첫 줄
2. 기존 `문서곡괭이` 설정의 같은 항목 검색키워드 첫 줄
3. 컬럼명

이 기능은 입력값 자체가 아니라 공고문에서 찾을 기준 키워드를 사용한다.

## 상단표 연결

기존 연결 기능은 유지한다.

- 상단 선택 row의 컬럼 더블클릭 -> 우측 패널의 같은 컬럼으로 이동
- 우측 입력칸 포커스 -> 상단 선택 row의 같은 컬럼으로 가로 스크롤
- 연결된 컬럼과 입력칸은 파란색으로 강조

## 주의

- 이전 프로젝트 문서의 `RightPane`, `DetailField`, `fieldSpecs` 구조는 참고용이다.
- 현재 구현은 `App.tsx` 안의 `NoticeDetailModal`에 있으므로 그대로 복사하지 않고 동작만 재해석한다.
- 상세모달 구조를 더 쪼개는 작업은 기능 안정 후 별도 리팩토링으로 진행한다.
