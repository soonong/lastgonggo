# 공고확인·특수실적·특수실적_공통 질문 목록

작성일: 2026-05-24

## 목적

`공고확인`, `특수실적`, `특수실적_공통`은 서로 합치지 않고 각각 따로 관리한다.
다만 문서곡괭이, 사람입력, 적격심사 2차 매핑과 모두 연결되므로 구현 전에 세부 규칙을 질문으로 고정한다.

답변 방식:

```text
질문 번호 기준으로 짧게 답변해도 된다.
예: C-01 = 공통이면 전체 적용, 특정 발주처가 있으면 특정 발주처 우선.
```

주의:

```text
아래 질문은 모두 예시를 기준으로 판단한다.
질문에 나온 예시가 틀린 방향이면 예시를 고쳐서 답변하면 된다.
```

# 1. 세 파일 공통 질문

## C-01. `발주처=공통`과 특정 발주처가 동시에 맞으면 우선순위는?

예시:

```text
공고 발주처=한국전력공사

공고확인 탭:
1) 발주처=공통, 결과값=현장확인
2) 발주처=한국전력공사, 결과값=현장확인

둘 다 검색키워드가 맞으면
특정 발주처 행을 우선할까?
아니면 둘 다 후보로 남길까?
```

## C-02. `입찰방식=빈값`은 전체 입찰방식 적용인가?

예시:

```text
특수실적 행:
입찰방식=빈값
결과값=사회적기업

공고 입찰방식=적격심사, PQ, 수의계약 중 무엇이든
이 행을 후보로 볼까?
```

## C-03. `검색키워드=빈값`인 행은 자동 파싱 대상인가, 사람 선택 목록인가?

예시:

```text
특수실적 행:
결과값=공연장[3000M]_준공금액
검색키워드=빈값

문서곡괭이가 공고문에서 `공연장`, `3000M`, `준공금액`을 자동으로 검색해야 하나?
아니면 이 행은 사람이 선택할 수 있는 목록으로만 둘까?
```

## C-04. `검색키워드` 여러 줄은 OR 조건인가?

예시:

```text
검색키워드:
전자시담
전자 시담
시담 방식

본문에 셋 중 하나만 있어도 결과값=전자시담으로 볼까?
```

## C-05. `제외키워드`는 검색키워드 근처에서만 볼까, 본문 전체에서 볼까?

예시:

```text
검색키워드=상호진출
제외키워드=허용하지 않음

본문 앞부분:
상호진출 허용

본문 뒷부분:
본 공사는 다른 조건에 대해서는 허용하지 않음

이 경우 `허용하지 않음`이 멀리 떨어져 있어도 제외할까?
아니면 검색키워드 주변 문장/문단 안에 있을 때만 제외할까?
```

## C-06. 같은 탭 안에서 `결과값`이 중복되면 허용할까?

예시:

```text
특수실적 탭:
1) 종목=전기, 발주처=공통, 결과값=의료시설[전기]_준공면적
2) 종목=전기, 발주처=한국전력공사, 결과값=의료시설[전기]_준공면적

결과값은 같지만 조건이 다르면 정상으로 볼까?
아니면 중복 경고를 띄울까?
```

## C-07. 한 공고에서 여러 결과값이 동시에 나오면 구분자는?

예시:

```text
공고문에서
공고확인=전자시담
공고확인=참가서류제출
두 개가 동시에 잡힘

최종 공고확인 컬럼은
전자시담/참가서류제출
처럼 `/`로 합칠까?
아니면 태그 배열로 따로 보관할까?
```

## C-08. 여러 결과값 순서는 무엇을 기준으로 할까?

예시:

```text
공고문에는 `참가서류제출` 문구가 먼저 나오고
뒤에 `전자시담` 문구가 나옴

CSV id는 전자시담이 더 작음

최종 표시는
참가서류제출/전자시담 처럼 본문 순서일까?
전자시담/참가서류제출 처럼 설정 id 순서일까?
```

## C-09. `대분류`는 매칭 조건인가, 화면 필터인가?

예시:

```text
특수실적 행:
대분류=건축물
결과값=공연장[3000M]_준공금액

공고문에서 공연장 실적이 나오면
대분류는 단순히 설정 화면 필터로만 쓰나?
아니면 공고의 다른 값과 `대분류=건축물`이 맞아야 후보로 보나?
```

## C-10. `메모`는 사람이 읽는 설명만 할까, 로직에서 읽을까?

예시:

```text
메모=한전 배전 기준일 때만 사용

이 문장을 코드가 조건으로 해석하면 안 되는지,
아니면 나중에 로직 조건으로도 쓸 수 있는지?
```

## C-11. `id`는 가져오기 때 유지해야 하나, 새로 발급해도 되나?

예시:

```text
공고확인 CSV에서 id=40, 결과값=전자시담

웹 설정으로 가져온 뒤에도 id=40을 유지해야
기존 문서곡괭이 근거 `rule_id=40`과 연결될까?
아니면 내부 DB id는 새로 만들고 원본 id를 별도 보관할까?
```

## C-12. 행 삭제와 비활성화는 구분할까?

예시:

```text
결과값=민간발주를 당장은 쓰지 않음

완전 삭제하면 과거 공고 근거가 끊길 수 있다.
이럴 때 `사용여부=false`로 두고 문서곡괭이만 제외하는 방식이 필요할까?
```
