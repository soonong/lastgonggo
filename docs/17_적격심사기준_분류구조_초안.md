# 적격심사기준 분류 구조 초안

작성일: 2026-05-24

## 목적

이 문서는 참고자료의 흐름을 그대로 복사하지 않고,
우리 공고분류 시스템에 맞는 적격심사기준 분류 구조를 다시 잡기 위한 초안이다.

핵심 목적은 하나다.

```text
공고 1건이 들어왔을 때 A4 적격심사세부기준 API를 호출할 수 있는
최종 적격평가기준_세부를 보수적으로 찾는다.
```

참고자료는 아래 문서에 따로 정리한다.
참고자료의 함수명, 파일명, 화면 경로는 새 구조의 고정값이 아니다.

```text
docs/19_적격평가기준_세부_참고사항_정리.md
```

## 참고사항 사용 원칙

참고자료에서 가져올 것은 운영 원칙이고, 그대로 고정하지 않을 것은 구현 세부명이다.

| 구분 | 가져올 것 | 그대로 고정하지 않을 것 |
|---|---|---|
| 흐름 | 1차 매칭 후 2차 매칭 | `applyPrimaryMatch`, `applySecondaryMatch` 같은 함수명 |
| 기준표 | 발주처/건설유형/금액/시행일 기준 | `master_evaluationCriteria.json` 파일명 |
| 2차 규칙 | 추가 조건으로 원발주처/세부기준 덮어쓰기 | `orgCriteria` 이름 |
| 제외 처리 | 정상 제외와 미매칭 구분 | `rules_적격_skipKeywords.json` 파일명 |
| 디버깅 | 실패 사유와 처치 방법 기록 | `_v1_...` 컬럼명 고정 |

## 확정 기준

아래 항목은 기존 대화에서 확정된 기준이다.

| 항목 | 기준 | 예시 |
|---|---|---|
| 금액 비교 | 추정가격 사용 | `추정가격=4.5억`, `기초금액=5.1억`이면 4.5억으로 기준표 선택 |
| 금액 경계 | 미만 | `2억미만`은 `199,999,999`까지 적용, `200,000,000`은 다음 구간 |
| 시행일 | 공고일 또는 입력일과 비교 | `시행일 <= 입력일`인 기준만 후보 |
| 복합 종목 | 현재는 주공종 기준 | `(건축,전기)`이면 주공종 `건축` 기준 |
| 적격심사기준 표기 | 원문 보존 | `별표5`, `별표-5`, `50억미만` 그대로 유지 |
| 적격평가기준_세부 | `원발주처_적격심사기준` | `행자부_별표-4` |

## 설계 방향

적격심사기준 분류는 한 덩어리 함수가 아니라 단계별 파이프라인으로 둔다.

```text
입력 공고
  -> 기준대상 판정
  -> 선행값 검증
  -> 발주처/원발주처 정규화
  -> 1차 기준 매칭
  -> 2차 세부조건 매칭
  -> 결과 검증
  -> A4 호출
  -> 디버그/처치정보 기록
```

## 0단계: 기준대상 판정

입찰방식에 따라 적격심사기준을 찾지 않아도 되는 공고를 먼저 걸러낸다.

이 단계의 출력은 `대상`, `제외`, `확인필요` 중 하나로 둔다.

예시:

```text
입찰방식=수의계약
  -> 제외
  -> 적격평가기준_세부=빈값
  -> 오류 아님
```

예시:

```text
입찰방식=적격심사
  -> 대상
  -> 적격평가기준_세부를 찾아야 함
```

## 1단계: 발주처/원발주처 정규화

수집된 발주처 문자열을 기준으로 실제 평가에 사용할 `원발주처` 후보를 만든다.

이 단계는 기존 `적격발주처 변경` 마스터와 연결된다.

예시:

```text
수집 발주처=한국남부발전 주식회사
적격발주처=한국남부발전
정규화 규칙=한국남부발전 -> 조달청
  -> 원발주처 후보=조달청
```

## 1.5단계: 선행값 검증

1차 기준 매칭 전에 필요한 값이 있는지 확인한다.
값이 없으면 무리해서 기준을 찾지 않고 미매칭 사유를 남긴다.

필수 선행값:

| 값 | 필요한 이유 | 없을 때 |
|---|---|---|
| 입찰방식 | 적격심사 대상 여부 판단 | 확인 필요 |
| 일반_기타_전문 | 1차 기준표의 건설유형 컬럼 선택 | 확인 필요 |
| 입력일 또는 공고일 | 시행일 기준 선택 | 확인 필요 |
| 추정가격 | 금액 구간 선택 | 확인 필요 |
| 원발주처 후보 | 1차 기준표 발주처 필터 | 확인 필요 |

예시:

```text
입찰방식=적격심사
일반_기타_전문=빈값
  -> 1차 매칭 중단
  -> 사유=건설유형 없음
  -> 처치=종목 분류 또는 사람 입력으로 일반_기타_전문 보완
```

## 2단계: 1차 기준 매칭

1차 기준 매칭은 기본 적격심사기준을 찾는 단계다.

마스터 이름은 구현에서 바뀔 수 있지만, 구조는 아래 컬럼을 가진다.

| 컬럼 | 의미 | 예시 |
|---|---|---|
| id | 행 식별값 | `39` |
| 원발주처 | 실제 평가 기준 발주처 | `한국전력공사` |
| 적격발주처 | 수집/홈페이지 기준 발주처 | `한국전력공사` |
| 건설유형 | 주공종 기준 건설유형 | `일반건설` |
| 시행일 | 기준 적용 시작일 | `2026-01-31` |
| 금액상한 | 미만 기준 금액 | `300000000` |
| 적격심사기준 | 원문 기준명 | `3억미만` |
| 사용여부 | 비활성화 가능 | `true` |
| 메모 | 사람 설명 | `한전 일반 기준` |

매칭 순서:

```text
1. 원발주처 후보와 건설유형으로 행을 좁힌다.
2. 시행일 <= 입력일 또는 공고일인 행만 남긴다.
3. 가장 최근 시행일 묶음을 선택한다.
4. 금액상한을 내림차순으로 보며 추정가격 < 금액상한이면 후보를 갱신한다.
5. 더 이상 미만 조건이 아니면 멈춘다.
6. 최종 후보의 적격심사기준으로 1차 키를 만든다.
```

예시:

```text
원발주처=한국전력공사
건설유형=일반건설
추정가격=250,000,000
입력일=2025-03-01

10억미만 -> 후보
3억미만 -> 후보 갱신
1억미만 -> 조건 불일치, 멈춤

1차 키=한국전력공사_3억미만
```

## 3단계: 2차 세부조건 매칭

2차 매칭은 1차 키만으로 부족한 특수 기준을 덮어쓰기 위한 단계다.

예를 들어 같은 `한국전력공사_3억미만`이라도
공고확인 값이 `배전공가`이면 실제 기준이 달라질 수 있다.

첨부된 `적격심사기준_변경_2026-05-24.csv`를 보면
2차 매칭은 `공고확인` 하나만 보는 구조가 아니라
A~H 조건 영역을 함께 보는 구조다.

실제 파일 컬럼:

| 구분 | 컬럼 |
|---|---|
| A | `id` |
| B~H 조건 | `적격평가기준_세부 (매칭키)`, `입찰방식`, `종목`, `공고확인`, `특수실적`, `등급공사`, `동일실적평가여부` |
| I~K 핵심 덮어쓰기 | `원발주처`, `적격발주처`, `적격평가기준_세부` |
| L 이후 추가 덮어쓰기 | `지역제한`, `추정가격기준`, `추정금액기준`, `기초금액기준`, 이후 추가 가능 |
| 관리 | `메모`, `편집`, `우선순위`, `사용여부` |

웹 관리용 권장 구조:

| 컬럼 | 의미 | 예시 |
|---|---|---|
| id | 행 식별값 | `1` |
| 조건_매칭키 | 1차 매칭 결과 | `한국전력공사_3억미만` |
| 조건_입찰방식 | 찾을 조건 | `적격심사` |
| 조건_종목 | 찾을 조건 | `전기` |
| 조건_공고확인 | 찾을 조건 | `배전공가` |
| 조건_특수실적 | 찾을 조건 | `무정전_준공금액` |
| 조건_등급공사 | 찾을 조건 | `1` |
| 조건_동일실적평가여부 | 찾을 조건 | `1` |
| 덮어쓰기_원발주처 | I~K 핵심 덮어쓰기 | `한국전력공사(배전공가)` |
| 덮어쓰기_적격발주처 | I~K 핵심 덮어쓰기 | `한국전력공사` |
| 덮어쓰기_적격평가기준_세부 | I~K 핵심 덮어쓰기 | `한국전력공사(배전공가)_3억미만` |
| 추가덮어쓰기_* | L 이후 확장 덮어쓰기 | `지역제한=1`, `추정가격기준=1` |
| 우선순위 | 여러 규칙 충돌 시 순서 | `10` |
| 사용여부 | 비활성화 가능 | `true` |
| 메모 | 사람 설명 | `공고확인에 배전공가가 있을 때` |

현재 파일에는 `우선순위`, `사용여부`가 없으므로
웹 설정 테이블로 올릴 때 추가하는 편이 안전하다.

조건 영역의 값이 비어 있으면 그 조건은 비교하지 않는 기본 규칙으로 볼 수 있다.

예시:

```text
1차키=조달청_별표-4
조건_공고확인=빈값
  -> 조달청_별표-4 기본 규칙
```

다만 실제 파일에는 빈 조건이 많으므로,
빈 조건을 모두 전체 통과로 볼지 여부는 사용자 확인 후 확정한다.

예시:

```text
매칭키=국방부_별표-3-유형1
입찰방식=PQ
공고확인=고난도
동일실적평가여부=빈값
  -> 동일실적평가여부는 보지 않는 조건인가?
```

첨부 파일 분석:

```text
docs/18_적격심사기준_2차매핑표_분석.md
```

## 4단계: 결과 검증

결과는 단순 문자열 하나가 아니라 상태와 이유를 함께 보관한다.

권장 출력:

| 필드 | 의미 | 예시 |
|---|---|---|
| 상태 | `found`, `skipped`, `missing`, `needs_review` | `found` |
| 1차키 | 1차 매칭 결과 | `한국전력공사_3억미만` |
| 원발주처 | 최종 원발주처 | `한국전력공사(배전공가)` |
| 적격심사기준 | 최종 기준명 | `3억미만` |
| 적격평가기준_세부 | A4 호출 키 | `한국전력공사(배전공가)_3억미만` |
| 선택이유 | 사람이 볼 설명 | `공고확인=배전공가 2차 규칙 적용` |
| 매칭근거 | 사용한 마스터 행 id | `1차:39, 2차:4` |
| 처치방법 | 실패 시 사람이 고칠 방법 | `추정가격 입력 필요` |

값 표현:

| 값 | 의미 | 표시 |
|---|---|---|
| 빈문자열 | 정상 제외 | 흰색 |
| null | 찾아야 하는데 못 찾음 | 확인색 또는 빨간 셀 |
| 문자열 값 | 매칭 성공 | 흰색 |

예시:

```text
입찰방식=적격심사
원발주처=미등록발주처
  -> 상태=missing
  -> 빨간 셀
  -> 미등록 발주처 목록에 추가
```

실패 사유별 처치:

| 사유 | 처치 | 예시 |
|---|---|---|
| 원발주처 미등록 | 1차 기준표에 발주처 기준 행 추가 | `경기도청` 기준 추가 |
| 적용 단계 없음 | 1차 기준표에 금액 단계 추가 | `30억미만` 행 추가 |
| 건설유형 없음 | 종목 분류 또는 사람 입력 보완 | `일반_기타_전문=기타건설` |
| 추정가격 없음 또는 0 | 추정가격 입력 | `추정가격=250000000` |
| 입찰방식 없음 | 입찰방식 입력 | `입찰방식=적격심사` |

## 실행 시점

실행 시점은 두 번으로 나눈다.

```text
1. AI 수집/문서곡괭이 이후 자동 실행
2. 사람이 공고확인 등 추가값을 입력하고 2차분류 완료를 누를 때 재실행
```

같은 입력으로 여러 번 실행해도 같은 결과가 나와야 하므로 멱등성을 지킨다.

주의:

```text
사람이 공고확인=배전공가를 입력한 뒤 2차분류 완료를 여러 번 눌러도
원발주처와 적격평가기준_세부가 계속 같은 값이어야 한다.
```

## 관리 화면 구조

`룰관리` 또는 `분류규칙` 안에 적격심사 기준 관리를 별도 그룹으로 둔다.

권장 구조:

```text
룰관리
  - 적격심사기준
    - 기준대상/제외 입찰방식
    - 발주처 정규화
    - 1차 기준표
    - 2차 세부조건
    - 미등록 발주처
```

참고자료의 파일명은 구현 후보로만 본다.

| 참고명 | 우리 구조에서의 위치 |
|---|---|
| `rules_적격_skipKeywords.json` | 기준대상/제외 입찰방식 |
| `적격발주처 변경` | 발주처 정규화 |
| `master_evaluationCriteria.json` | 1차 기준표 |
| `orgCriteria` | 2차 세부조건 |

## 충돌 정리

기존 문서와 참고자료 사이의 충돌 후보는 1개다.

```text
기존 원칙: 전처리 기준 파일은 웹 설정에서 추가/변경/수정 가능해야 한다.
참고자료: 1차 기준표는 data/master_evaluationCriteria.json 직접 편집이라고 되어 있다.
```

정리 방향:

```text
구현 초기에는 JSON/CSV 가져오기 또는 직접 편집을 허용할 수 있다.
최종 웹 구조에서는 1차 기준표도 설정 화면에서 편집 가능해야 한다.
```

## 남은 결정

아래는 구조 설계에서 더 확인해야 할 항목이다.

1. 제외 입찰방식 목록을 어디까지 둘지

예시:

```text
수의계약, 최저가, 종심제 중 어떤 값이 A4 호출 제외 대상인가?
```

2. 2차 세부조건의 조건 컬럼을 고정할지, 동적으로 둘지

예시:

```text
조건컬럼=공고확인 하나만 쓸지,
공고확인 + 특수실적 + 종목 같은 여러 컬럼을 선택할 수 있게 할지
```

3. 미등록 발주처 알림을 배치 종료 후 한 번에 띄울지

예시:

```text
미등록 발주처=경기도청, 인천시청
  -> 수집 완료 후 한 번에 알림
```
