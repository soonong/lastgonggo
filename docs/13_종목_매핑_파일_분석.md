## 요약

이 파일은 원본 공고문이나 등록증에 적힌 자격 표현을 표준 종목 정보로 바꾸는 매핑 마스터다.

업무 용어는 `종목`으로 통일한다.
단, CSV 원본 컬럼명이 아래처럼 되어 있으므로 파일을 설명할 때만 원본 컬럼명을 병기한다.

| CSV 원본 컬럼 | 업무 해석 |
|---|---|
| 업종 | 종목 |
| 주력업종 | 주력종목 |
| 상위업종 | 상위종목 |
| 대체업종 | 대체종목 |

예시:

```text
원본 `일반소방시설공사업(기계)(업종코드:0038)`
  -> 일반_기타_전문=`기타건설`
  -> 종목=`기계소방`
  -> 주력종목=`기계소방`
  -> 상위종목=`전문소방`
```

예시:

```text
원본 `금속구조물 · 창호 · 온실공사`
  -> 일반_기타_전문=`전문건설`
  -> 종목=`금속창호지붕`
  -> 주력종목=`금속구조물`
  -> 단독평가종목 후보=`금속창호지붕_금속구조물`
```

## 확정 반영할 규칙

전문건설에서 `종목`과 `주력종목`이 다르면 단독평가종목은 `종목_주력종목` 형태로 만든다.

예시:

```text
종목=금속창호지붕
주력종목=금속구조물
  -> 단독평가종목=금속창호지붕_금속구조물
```

전문건설에서 `종목`과 `주력종목`이 같으면 주력종목은 없는 것으로 보고 삭제한다.

예시:

```text
종목=철콘
주력종목=철콘
  -> 주력종목 삭제
  -> 단독평가종목=철콘
```

`종목`은 사용자 표시용이고, `단독평가종목`은 실제 적격심사 계산 기준이다.

예시:

```text
공고문: 금속창호지붕(주력분야: 금속구조물)
종목: 금속창호지붕
단독평가종목: 금속창호지붕_금속구조물
종목1: 금속창호지붕_금속구조물
```

## 주요 패턴

### 긴 키워드 우선 매칭

`길이`는 `원본 (등록증 표기)`의 실제 문자열 길이와 모두 일치한다.
따라서 매칭 엔진은 파일 순서를 그대로 믿지 않고, 실행 시 `길이` 내림차순으로 정렬한 뒤 매칭한다.

예시:

```text
공고문에 `일반소방시설공사업(기계)`가 있으면
짧은 `소방`보다 긴 `일반소방시설공사업(기계)`를 먼저 매칭해야
종목=`기계소방`으로 잡을 가능성이 높다.
```

### 상위종목

상위종목은 현재 두 종류가 확인된다.

| 상위종목 | 행 수 | 의미 |
|---|---:|---|
| 전문소방 | 12 | 기계소방/전기소방의 상위 인정 종목 |
| 토건 | 8 | 건축/토목의 상위 인정 종목 |

예시:

```text
건축 -> 건축/토건
토목 -> 토목/토건
건축/토목 -> (건축,토목)/토건
```

예시:

```text
기계소방 -> 기계소방/전문소방
전기소방 -> 전기소방/전문소방
기계소방/전기소방 -> (기계소방,전기소방)/전문소방
```

### 대체종목

대체종목은 현재 `가스난방`과 `기계가스설비` 계열에서 확인된다.
대체종목은 참여 가능 면허 매칭에서 자동 인정한다.
표시용 종목에는 원 종목과 대체종목을 슬래시로 함께 표시한다.
단독평가종목은 원 종목 기준을 유지한다.

| 대체종목 | 행 수 | 예시 |
|---|---:|---|
| 가스난방 | 52 | `기계가스설비` 계열의 대체 묶음 |
| 기계가스설비 | 28 | `가스난방` 계열의 대체 묶음 |

예시:

```text
`기계설비·가스설비공사업`
  -> 종목=`기계가스설비`
  -> 대체종목=`가스난방`
  -> 회사 면허가 `가스난방`이면 매칭 가능으로 본다
  -> 표시용 종목=`기계가스설비/가스난방`
```

### 상호진출_상대종목

상호진출 상대종목은 종목매핑 마스터에서 함께 관리하는 것이 좋다.
1차 컬럼명은 `상호진출_상대종목`으로 둔다.

예시:

```text
일반종목=건축
상호진출_상대종목=실내건축, 금속창호지붕, 도장습식방수석공, 비계구조, 철콘, 지반조성포장
```

확정된 1차 매핑:

| 일반종목 | 상호진출_상대종목 |
|---|---|
| 건축 | 실내건축, 금속창호지붕, 도장습식방수석공, 비계구조, 철콘, 지반조성포장 |
| 토목 | 지반조성포장, 상하수도, 철도궤도, 수중준설, 승강기삭도, 철콘, 비계구조 |
| 산업환경설비 | 기계가스설비, 상하수도, 수중준설, 가스난방 |
| 조경 | 조경식재시설물, 지반조성포장 |

### 미분류

`미분류`가 129행 있다.
해외건설, 신재생, 환경전문, 광해방지, 개인하수처리, 나무병원 등 일반/전문/기타건설로 바로 넣기 어려운 항목들이 포함된다.
미분류는 시설 종목 매칭에서 제외한다.

예시:

```text
`해외건설업(전문건설업-도장공사업)`
  -> 일반_기타_전문=`미분류`
  -> 종목=`해외전문건설_도장습식방수석공`
  -> 주력종목=`도장`
  -> 시설 종목 매칭 제외
```

### 업종조건없음

`업종조건없음` 1행은 종목과 주력종목이 비어 있다.
이 행은 "종목 조건이 없는 공고"를 표현하는 특수값이다.
종목에는 `업종조건없음`을 그대로 넣지 않는다.
최종 반환값은 종목과 단독평가종목 모두 빈값이다.

예시:

```text
원본=`업종조건없음`
일반_기타_전문=`미분류`
종목=``
주력종목=``
단독평가종목=``
```

## 중복과 충돌

원본 표현이 중복된 그룹은 27개다.
그중 매핑 결과가 서로 다른 충돌 그룹은 10개다.

충돌 예시:

| 원본 | 매핑 A | 매핑 B |
|---|---|---|
| 조경시설물 | `조경시설물` | `조경식재시설물 / 주력종목=조경시설물` |
| 기계설비 | `기계설비` | `기계가스설비 / 대체종목=가스난방` |
| 소방 | `소방` | `전문소방` |
| 가스 | `가스` | `가스난방 / 대체종목=기계가스설비` |
| 포장 | `포장` | `지반조성포장 / 주력종목=포장` |
| 토공 | `토공` | `지반조성포장 / 주력종목=토공` |

해소 원칙:

```text
동일한 원본 키워드가 서로 다른 결과로 매핑되더라도,
원본문자 하나만 보고 확정하지 않는다.
먼저 수집/파싱 단계에서 `종목`, `전문건설_주력분야`, `일반_기타_전문`을 만든 뒤
정규화된 값을 기준으로 종목묶기를 한다.
```

예시:

```text
공고문=`포장공사업`
  -> 종목=지반조성포장
  -> 전문건설_주력분야=포장
  -> 단독평가종목=지반조성포장_포장
```

예시:

```text
공고문=`포장공사업/토공사업`, 종목모두보유=0
  -> 종목=지반조성포장
  -> 단독평가종목=지반조성포장_포장/지반조성포장_토공

공고문=`포장공사업/토공사업`, 종목모두보유=1
  -> 종목=지반조성포장
  -> 단독평가종목=(지반조성포장_포장,지반조성포장_토공)
```

예시:

```text
공고문=`소방`
  -> 전처리에서 전문소방으로 변환
  -> `소방`과 `전문소방` 중 고르는 충돌은 종목묶기 단계에서 발생하지 않는다.
```

## 종목묶기와의 관계

이 파일은 원본 표현을 표준 종목 객체로 바꾸는 입력 마스터다.
괄호, 쉼표, 슬래시, 상호진출, 상위종목 확장, 단독평가종목 생성은 `docs/rules/종목묶기룰.md`에서 처리한다.

예시:

```text
원본 `토공/토목`
  -> 매핑: 토공=전문건설, 종목=지반조성포장, 주력종목=토공
  -> 매핑: 토목=일반건설, 종목=토목, 상위종목=토건
  -> 종목묶기룰에서 `지반조성포장 / 토목 / 토건` 형태로 조립
```

## 설명이 필요한 지점

### 상호진출_상대종목 컬럼 추가 방식

종목매핑 파일에 `상호진출_상대종목` 컬럼을 추가한다.
별도 상호진출 테이블로 분리하지 않는다.

예시:

```text
일반종목=건축
상호진출_상대종목=실내건축,금속창호지붕,도장습식방수석공,비계구조,철콘,지반조성포장
```
