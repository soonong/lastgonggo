# 곡괭이전나라시 특수조건 라우팅 분리 (2026-05-30)

## 결정

`특수조건` 값을 `공고확인`, `특수실적`, `특수실적_공통`으로 복사하는 라우팅은 곡괭이전나라시에서 실행하지 않는다.

## 이유

곡괭이전나라시는 서버공고 raw만 보고 가능한 정리 단계다. 특수조건 라우팅은 문서곡괭이가 공고문에서 조건값을 찾은 뒤 적용하거나, 사람이 입력한 값을 저장/2차분류할 때 적용하는 규칙이다.

## 구현

- `preprocessRows`에 `skipSpecialConditionRouter` 옵션을 추가했다.
- 입찰공고 원본 탭의 `곡괭이전나라시` 호출에서는 이 옵션을 켠다.
- 문서곡괭이 이후 전처리, 공고관리 재전처리, 2차분류 호출에서는 기존처럼 라우팅을 유지한다.

## 예시

```text
입찰공고 원본 > 곡괭이전나라시
특수조건=사회적기업
-> 공고확인/특수실적/특수실적_공통으로 복사하지 않음

문서곡괭이 이후 또는 사람 입력 저장/2차분류
특수조건=사회적기업
-> 설정 결과값과 일치하면 특수실적_공통=사회적기업으로 복사 가능
```
