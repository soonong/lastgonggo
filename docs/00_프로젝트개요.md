# 프로젝트 개요

## 목적

서버공고 API에서 받은 입찰공고 전체 컬럼을 표준 형식으로 전처리하고,
빈 값은 공고문 파싱과 사람 검증으로 보완한 뒤 최종 엑셀 파일로 출력한다.

## 기본 흐름

```text
수집 -> 전처리 -> 사람입력 -> 최종출력
                  ^
                룰관리
```

## 핵심 원칙

| 원칙 | 내용 | 예시 |
|---|---|---|
| 원본 보존 | 서버공고 API에서 받은 원본은 덮어쓰지 않는다. A3/A4 같은 보조 API 결과도 출처를 분리해 보관한다. | 서버공고 `공고명` 원본값은 그대로 보관하고 표준 `공고명`만 따로 만든다. |
| 사람 입력 우선 | 사람이 입력하거나 검증한 값은 최우선으로 사용한다. | 사람이 `업종=전기공사업`으로 수정하면 API 값 `기타`가 다시 덮어쓰면 안 된다. |
| 보수적 처리 | 확실하지 않은 값은 억지로 채우지 않고 `검토필요`로 남긴다. | `수도권 업체 우대`는 `지역제한=서울`로 확정하지 않는다. |
| 근거 기록 | 처리 방법, 선택 이유, 출처, 예시를 남긴다. | `지역제한=서울`, 출처 `공고문`, 이유 `서울특별시 소재 업체 문구 확인`. |
| 페이지 최소화 | 발주처, 업무유형, 종목별 페이지를 만들지 않고 필터와 룰로 처리한다. | `나라장터 > 시설 > 전기`는 페이지가 아니라 필터 조건이다. |

## 값 선택 기본 우선순위

```text
사람 입력값 > 코드 규칙값 > 공고문에서 명확히 추출한 값 > AI 후보값
```

AI 후보값은 확실한 근거가 있을 때만 사용하고, 애매하면 `검토필요`로 남긴다.
