# 상세모달 종목 탭·동일실적 구현 메모 (2026-05-31)

## 목적

공고관리 상세모달의 우측 `종목` 탭을 일반 입력칸 모음이 아니라 종목 검수 전용 화면으로 분리한다.
기존 F12 문서는 그대로 복사하지 않고, 현재 프로젝트의 컬럼명과 `NoticeDetailModal` 구조에 맞춰 기능만 재해석했다.

## 적용 범위

- 파일: `app/src/jongmokDetail.tsx`
- 연결: `app/src/App.tsx` 의 `NoticeDetailModal`
- 스타일: `app/src/App.css`

## 5번: 종목 탭

종목 탭 상단에는 다음 필드를 편집한다.

- `일반_기타_전문`
- `종목`
- `전문건설_주력분야`
- `단독평가종목`
- `종목_모두보유`

`종목`, `전문건설_주력분야`, `종목_모두보유`에서 Enter 또는 blur가 발생하면 기존 종목묶기 기능을 다시 실행한다.

종목묶기 실행 결과:

- `종목` 재생성
- `단독평가종목` 재생성
- `일반_기타_전문`, `전문건설_주력분야` 보강
- `단독평가종목`을 기준으로 `종목세부JSON` 재생성

`단독평가종목`을 사람이 직접 수정한 경우에는 종목묶기를 다시 돌리지 않고, 입력된 `단독평가종목`을 파싱해 `종목세부JSON`만 다시 만든다.

파싱 규칙:

- `/`는 평가 옵션 구분이다. 화면에서는 그룹 단위로 접고 펼친다.
- 괄호 안의 `,`는 같은 옵션에서 모두 필요한 묶음이다.
- 예: `a/(b,c)/d` → 화면 그룹은 `a`, `(b,c)`, `d`
- 저장되는 `종목세부JSON`은 문서 규칙대로 앞에서부터 `a`, `b`, `c`, `d` 순서의 평가종목 목록이다.
- 기존 `종목세부JSON`에 있던 종목별 입력값은 같은 종목명 기준으로 보존한다.

`종목세부JSON`은 단독평가종목의 평가종목 단위 목록으로 저장한다. `/`와 괄호 구조는 화면 렌더링에서 `단독평가종목`을 다시 해석해 보여준다.

화면 구분 방식:

- `/`로 나뉜 평가 옵션은 `평가 그룹` 단위로 접고 펼친다.
- 각 그룹 안의 개별 종목은 독립 카드로 보여준다.
- 종목 카드는 왼쪽 색상 라인, `종목 N` 번호칩, 동일실적 개수칩으로 구분한다.
- 카드 안에는 `기본 입력` 영역과 동일실적 입력 영역을 나눠 보여준다.
- 이 구분은 화면 표시용이며, 종목묶기 결과나 `종목세부JSON` 저장 규칙은 바꾸지 않는다.

## 6번: 동일실적 블록

각 종목 아이템 안에 동일실적을 붙일 수 있다.

기본 카테고리:

- `준공금액`
- `준공단위`

동일실적 1개는 다음 6개 필드를 가진다.

- `참가자격기준규모`
- `평가기준`
- `만점실적`
- `최소인정규모`
- `실적평가기간`
- `평가비율`

특수실적 설정 데이터의 `결과값`, `중분류`, `특수실적`, `이름` 중 존재하는 값을 자동완성 후보로 쓴다.
입력값에 `금액`이 포함되면 `준공금액`, 아니면 `준공단위`에 추가한다.

## 주의

- 이 구현은 기존 F12 컴포넌트 이름을 그대로 쓰지 않는다.
- 현재 앱에서는 `draftRow`에만 반영하고, `저장 / 닫기`를 눌러야 원 row에 저장된다.
- 종목묶기 규칙 자체는 `pipeline.ts`와 종목묶기룰 문서를 기준으로 한다. 상세모달 종목 탭은 그 기능을 실행하고 결과를 검수·수정하는 화면이다.
- 적격심사기준 API 조회와 기준 아코디언 표시는 `적격심사기준` 탭 책임이다. 종목 탭에 섞지 않는다.
