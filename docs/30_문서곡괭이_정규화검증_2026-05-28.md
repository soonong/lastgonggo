# 문서곡괭이 정규화 검증 구현 기록 (2026-05-28)

## 구현 범위

- `검수 > 공고문 정규화 검증` 탭을 추가했다.
- A3 파싱용공고문 HTML을 원문줄, 정규화 문단, 번호/섹션, 표 후보로 분해해 볼 수 있게 했다.
- 같은 화면에서 문서곡괭이 파싱 결과와 근거를 함께 확인한다.
- `문서곡괭이_룰.csv`를 운영 엑셀 형식 기준으로 갱신했다.
- `종목매핑 > 종목묶기규칙` 탭을 추가해 종목/단독평가종목/종목세부JSON 예시를 표로 관리한다.

## 정규화 방어 기준

- `td/th/tr/p/div/br/li/h*`를 문서 경계로 보고 줄을 만든다.
- 줄바꿈 때문에 끊어진 문장은 앞줄이 종결되지 않았고 새 번호가 아니면 같은 문단으로 합친다.
- `1.`, `1)`, `가.`, `①`, `제1조` 같은 번호 시작은 새 문단/섹션으로 본다.
- 표처럼 보이는 줄은 `|` 기준으로 가로형/복합형 후보와 세로형 후보를 구분한다.
- 본문이 이미지뿐이면 텍스트 추출 실패로 숨기지 않고 `이미지형 공고문` 문단과 경고를 남긴다.

## 테스트

사용자가 준 62개 공고번호를 `/api/parser/normalize`로 배치 검증했다.

- 총 62건
- 성공 62건
- 실패 0건
- 경고 2건

경고 2건은 텍스트가 없는 이미지형 공고문이다.

- `R26BK01527282-000`
- `R26BK01544276-000`

결과 파일:

- `data/generated/parser_normalization_batch_2026-05-28.csv`

## 다음 점검 포인트

- 이미지형 공고문을 OCR로 보강할지, 사람 확인으로 둘지 결정이 필요하다.
- 표 후보는 현재 HTML 변환 결과의 셀 구분을 기준으로 감지한다. 실제 표별 추출 룰은 조건판단형태와 함께 추가 검증한다.
- 문서곡괭이 룰 중 조건판단형태가 비어 있는 컬럼은 사용자가 설명하면 룰로 등록한다.

## 2026-05-28 검수탭 정리

- 설정 화면은 `data/settings/설정탭구조.csv`의 `화면유형`을 기준으로 이동한다.
- 현재 검수탭은 `문서곡괭이 검증`, `조건판단형태 가이드`, `공고문 정규화 검증`, `섹션설정` 4개만 운영 대상으로 둔다.
- 예전 `activeMenu` 직접 분기였던 `문서곡괭이 룰`, `문서곡괭이 리포트`, `샘플 공고 테스트` 화면 코드는 제거했다.
- 기존 용어 `파서` 호환 분기는 제거하고, 컬럼 우선순위 판단은 `문서곡괭이` 명칭만 기준으로 한다.
- 문서곡괭이 검증 뉴버전은 기존 구현을 복사하지 않는다. 현재 정규화/조건판단형태/섹션설정 데이터는 참고하되, 사용자가 확정한 구조대로 새 화면과 실행 흐름을 만든다.

## 2026-05-28 신문서곡괭이 1차 화면

- 기존 `문서곡괭이 검증` 탭은 그대로 두고 `검수 > 공고문 파싱 가이드 > 신문서곡괭이` 탭을 추가했다.
- 화면 컬럼은 `항목`, `조건판단형태`, `검색키워드`, `제외키워드`, `고정값`, `결과값`으로 시작한다.
- `항목`은 공고분류컬럼 설정의 표준 컬럼 129개를 기준으로 자동 생성한다.
- `결과값`은 저장하지 않는다. 상단 공고번호로 A3 파싱용공고문을 실행했을 때만 채워지는 검증값이다.
- 검색키워드의 여러 줄은 OR로 처리하고, `3_2`는 `(결과값:후보1/후보2)` 별칭 문법을 사용한다.
- 이번 1차 구현에서는 공고문 전체를 강하게 정규화하는 방식은 적용하지 않는다. 기존 문단 정규화와 룰 단위 키워드 매칭만 사용한다.
