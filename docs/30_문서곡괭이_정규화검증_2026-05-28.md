# 문서곡괭이 정규화 검증 구현 기록 (2026-05-28)

## 구현 범위

- `검수 > 공고문 정규화 검증` 탭을 추가했다.
- A3 파싱용공고문 HTML을 원문줄, 정규화 문단, 번호/섹션, 표 후보로 분해해 볼 수 있게 했다.
- 같은 화면에서 문서곡괭이 파싱 결과와 근거를 함께 확인한다.
- `문서곡괭이_룰.csv`를 운영 엑셀 형식 기준으로 갱신했다.
- `종목매핑 > 종목묶기규칙` 탭을 추가해 종목/단독평가종목/종목세부JSON 예시를 표로 관리한다.

## 정규화 방어 기준

- `td/th/tr/p/div/br/li/h*`를 문서 경계로 보고 줄을 만든다.
- 줄바꿈 때문에 끊어진 문장은 앞줄이 종결되지 않았고 새 번호가 아니면 같은 문단으로 합친다.
- `1.`, `1)`, `가.`, `①`, `제1조` 같은 번호 시작은 새 문단/섹션으로 본다.
- 표처럼 보이는 줄은 `|` 기준으로 가로형/복합형 후보와 세로형 후보를 구분한다.
- 본문이 이미지뿐이면 텍스트 추출 실패로 숨기지 않고 `이미지형 공고문` 문단과 경고를 남긴다.

## 테스트

사용자가 준 62개 공고번호를 `/api/parser/normalize`로 배치 검증했다.

- 총 62건
- 성공 62건
- 실패 0건
- 경고 2건

경고 2건은 텍스트가 없는 이미지형 공고문이다.

- `R26BK01527282-000`
- `R26BK01544276-000`

결과 파일:

- `data/generated/parser_normalization_batch_2026-05-28.csv`

## 다음 점검 포인트

- 이미지형 공고문을 OCR로 보강할지, 사람 확인으로 둘지 결정이 필요하다.
- 표 후보는 현재 HTML 변환 결과의 셀 구분을 기준으로 감지한다. 실제 표별 추출 룰은 조건판단형태와 함께 추가 검증한다.
- 문서곡괭이 룰 중 조건판단형태가 비어 있는 컬럼은 사용자가 설명하면 룰로 등록한다.
