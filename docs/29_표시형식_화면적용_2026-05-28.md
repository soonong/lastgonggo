# 표시형식 화면 적용 (2026-05-28)

## 목적

`공고수집 > 공고분류컬럼 설정`의 `표시형식`은 데이터 저장값을 바꾸는 규칙이 아니라, 표와 상세 화면에서 사람이 보는 값을 일관되게 보여주는 표시 규칙이다.

예:

| 항목 | 표시형식 | 화면 표시 예 |
|---|---|---|
| 기초금액 | `###,###,###` | `169,730,000` |
| 추정가격 | `###,###,###` | `154,300,000` |
| 투찰율 | `##.###%` | `87.745%` |
| 입력일 | `yy-mm-dd` | `2026-05-15` |
| 등록마감일 | `yy-mm-dd h:mm` | `2026-05-28 18:00` |

## 적용 화면

같은 공통 표 렌더러를 쓰는 아래 화면에 동일하게 적용한다.

- 입찰공고 원본
- 입찰공고 전처리
- 공고관리
- 2차분류 완료
- 공고 상세 모달 우측 컬럼 값

## 원칙

- 원본 row 값은 변경하지 않는다.
- 화면 표시만 `표시형식`에 맞게 변환한다.
- 툴팁에는 표시값과 원본값을 함께 확인할 수 있게 한다.
- 검색, 정렬, 전처리 계산은 기존 저장값을 기준으로 유지한다.

## 지원 형식

- 금액/숫자: `###,###,###`, `#,###`, `0.00`, `##.##`
- 퍼센트: `##%`, `##.###%`, `###.##%`
- 날짜: `yy-mm-dd`, `yy-mm-dd h:mm`

표시형식이 비어 있거나 위 형식에 해당하지 않는 설명형 텍스트이면 원본 값을 그대로 보여준다.
