# 공고확인·특수실적·특수실적_공통 설정데이터 초안

작성일: 2026-05-24

## 목적

`공고확인`, `특수실적`, `특수실적_공통`은 사용자가 계속 추가, 수정, 삭제해야 하는 운영 데이터다.
세 데이터는 비슷한 컬럼을 가지고 있지만 의미와 쓰임이 다르므로 절대 하나로 합치지 않는다.

확정 원칙:

```text
공고확인        -> 독립 설정 탭, 독립 저장소
특수실적        -> 독립 설정 탭, 독립 저장소
특수실적_공통   -> 독립 설정 탭, 독립 저장소
```

이유:

```text
공고확인은 사용자에게 확인할 태그를 보여주는 데이터다.
특수실적은 종목/발주처별 실적 조건을 찾는 데이터다.
특수실적_공통은 기업인증, 신용평가, 장비, 협력업체 같은 공통 조건을 관리하는 데이터다.
셋을 합치면 검색 조건, 화면 표시, 2차 매핑 조건, 문서곡괭이 반환값이 섞인다.
```

예시:

```text
공고확인 결과값=전자시담
  -> 공고확인 컬럼에 들어간다.

특수실적 결과값=철도배전선로[154Kv]_준공금액
  -> 특수실적 컬럼에 들어간다.

특수실적_공통 결과값=사회적기업
  -> 특수실적_공통 또는 공통 조건 후보로 들어간다.

위 세 값을 한 테이블로 합치면 `전자시담`과 `사회적기업`이 같은 종류의 선택값처럼 보여 오류가 난다.
```

## 설정 화면 구조

권장 위치:

```text
룰관리
  - 확인/실적 키워드
    - 공고확인
    - 특수실적
    - 특수실적_공통
```

각 탭은 독립적으로 저장한다.
CSV 가져오기/내보내기도 각각 따로 한다.

예시:

```text
공고확인 탭에서 `전자시담`을 삭제해도
특수실적 탭의 `전기_준공금액` 데이터는 영향을 받지 않는다.

특수실적_공통 탭에서 `사회적기업` 검색키워드를 수정해도
공고확인 탭의 `민간발주` 검색키워드는 영향을 받지 않는다.
```

공통 편집 기능:

```text
행 추가
행 수정
행 삭제
CSV 가져오기
CSV 내보내기
사용여부 관리
키워드 테스트
변경 이력
```

## 2026-05-26 답변 반영 정정

아래 내용은 사용자 답변으로 정정한다.

```text
종목, 발주처, 대분류는 공고 row와 자동 매칭하기 위한 조건이 아니다.
사용자가 결과값을 찾기 위한 화면 검색·필터용 메타다.
```

예시:

```text
특수실적 행:
종목=전기
발주처=한국철도공사
대분류=철도신호
결과값=철도신호설비_준공금액

위 종목/발주처/대분류는 결과값 검색용이다.
row.종목=전기가 아니어도 검색키워드가 맞으면 문서곡괭이 후보가 된다.
```

확정된 공통 규칙:

| 항목 | 규칙 | 예시 |
|---|---|---|
| 검색키워드 빈값 | 자동 파싱하지 않고 사람이 직접 입력 | `공연장[3000M]_준공금액` |
| 검색키워드 여러 줄 | OR 조건 | `전자시담`, `전자 시담` |
| 제외키워드 | 검색키워드 매칭 문맥 안에서 비교 | 같은 문단의 `허용하지 않음` |
| 결과값 중복 | 같은 탭 안에서는 오류 | 공고확인 탭에 `전자시담` 2개 |
| 여러 결과값 | `/`로 연결 | `전자시담/기초금액발표전` |
| id | 가져올 때 새로 발급 | CSV id 그대로 고정하지 않음 |
| 메모 | 사람 설명용 | 로직에서 읽지 않음 |

상세 반영 문서:

```text
docs/24_공고확인_특수실적_답변반영.md
```

## 컬럼 해석

세 파일의 공통 컬럼은 이름이 같아도 같은 테이블로 합치라는 뜻이 아니다.
각 탭 안에서 같은 방식으로 해석할 수 있다는 의미다.

| 컬럼 | 해석 | 예시 |
|---|---|---|
| id | 탭 안에서의 행 식별값 | `13` |
| 종목 | 결과값 검색·필터용 메타. 공고확인/공통 탭의 값은 옛 구분값 | `전기`, `공고확인` |
| 발주처 | 결과값 검색·필터용 메타. `PQ`처럼 실제로는 입찰방식인 값도 있음 | `한국전력공사`, `PQ` |
| 입찰방식 | 서버수집 `입찰방식` 컬럼과 비교 가능한 조건 | `PQ` |
| 대분류 | 화면 필터와 묶음 기준 | `건축물`, `기업인증` |
| 결과값 | 실제 컬럼에 들어갈 운영 코드 | `배전공가`, `공연장[3000M]_준공금액` |
| 상위실적 | 사용자 검색용 메타. 문서곡괭이와 계산맨에서 사용하지 않음 | `공연장[3000M]_준공금액` |
| 검색키워드 | 문서곡괭이가 공고문에서 찾을 문구 | `사회적기업` |
| 제외키워드 | 매칭 제외 문구 | `해당없음` |
| 메모 | 사람이 읽는 설명 | `한전 배전 특수조건` |

`특수실적_공통`만 추가로 가진 컬럼:

| 컬럼 | 해석 | 예시 |
|---|---|---|
| 적격업체 | 사용자 검색용 메타. 문서곡괭이와 계산맨에서 사용하지 않음 | `true`, `false` |

## 저장 구조

세 데이터는 저장소도 분리한다.

```text
settings.confirmNotice       = 공고확인
settings.specialRecord       = 특수실적
settings.specialRecordCommon = 특수실적_공통
```

예시:

```text
settings.confirmNotice rows:
  결과값=전자시담

settings.specialRecord rows:
  결과값=철도배전선로[154Kv]_준공금액

settings.specialRecordCommon rows:
  결과값=사회적기업
```

## 문서곡괭이와 연결

문서곡괭이는 세 설정을 각각 따로 읽는다.
하나의 통합 목록으로 합쳐서 검색하지 않는다.

흐름:

```text
파싱용공고문 HTML
  -> 문서곡괭이 본문정규화
  -> 공고확인 설정으로 공고확인 후보 추출
  -> 특수실적 설정으로 특수실적 후보 추출
  -> 특수실적_공통 설정으로 특수실적_공통 후보 추출
  -> 각 결과를 서로 다른 컬럼 후보로 기록
```

예시:

```text
본문에 "전자시담" 있음
  -> 공고확인 설정에서만 검색
  -> 공고확인=전자시담

본문에 "철도배전선로 154Kv" 있음
  -> 특수실적 설정에서 검색
  -> 특수실적=철도배전선로[154Kv]_준공금액

본문에 "사회적기업" 있음
  -> 특수실적_공통 설정에서 검색
  -> 특수실적_공통=사회적기업
```

근거 기록:

```text
문서곡괭이는 결과값만 남기지 않고
어떤 설정 탭, 어떤 id, 어떤 검색키워드, 어떤 원문에서 뽑았는지 같이 기록한다.
```

예시:

```text
컬럼=공고확인
값=전자시담
설정탭=공고확인
rule_id=40
matched_keyword=전자시담
source_text=본 공고는 전자시담 방식으로 진행합니다.
```

## 2차 매핑과 연결

적격심사 2차 매핑표는 `공고확인`, `특수실적` 같은 실제 참조 컬럼을 본다.
조건값은 각 설정 탭의 `결과값`과 맞춘다.
`특수실적_공통`은 2차 매핑표가 직접 참조하지 않는다.
사람이 `특수조건`에 입력한 값은 계산맨/라우터가 필요한 경우 `공고확인` 같은 실제 참조 컬럼으로 옮긴다.

예시:

```text
2차 매핑 조건:
공고확인=배전공가

공고확인 설정:
결과값=배전공가
검색키워드=배전공가

문서곡괭이 결과:
공고확인=배전공가
  -> 2차 매핑 조건에 사용 가능
```

예시:

```text
2차 매핑 조건:
특수실적=무정전_준공금액

특수실적 설정:
결과값=무정전_준공금액
검색키워드=무정전

문서곡괭이 결과:
특수실적=무정전_준공금액
  -> 2차 매핑 조건에 사용 가능
```

## 검증 기준

각 탭은 독립적으로 검증한다.

| 검증 | 이유 | 예시 |
|---|---|---|
| 탭 안 결과값 중복 확인 | 같은 탭 안에서 같은 선택값이 여러 의미로 쓰이면 충돌 | 공고확인 탭에 `전자시담` 중복 |
| 탭 간 결과값 중복은 허용 가능 | 서로 다른 컬럼에 들어가므로 의미가 다를 수 있음 | 공고확인 `특수실적`과 특수실적_공통 `특수실적` |
| 검색키워드 빈값 표시 | 자동 파싱이 어려운 행을 사람이 알 수 있게 함 | `공연장[3000M]_준공금액` 키워드 빈값 |
| 2차 매핑 조건 존재 확인 | 2차 조건값이 설정에 없는지 확인 | `공고확인=배전공가`가 공고확인 탭에 있는지 |
| 예시본문 테스트 | 룰 품질 확인 | `전자시담` 본문에서 `전자시담` 추출 |

주의:

```text
검색키워드가 빈값인 행을 무조건 오류로 보지는 않는다.
빈값 행은 자동 파싱 대상이 아니며 사람이 직접 선택하거나 입력한다.
```
