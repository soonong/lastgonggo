# 설정 8대분류 정리

## 목적

설정 화면의 검은 좌측 메뉴와 흰색 설정 탭을 운영 기준에 맞춰 8개 대분류 중심으로 단순화한다.

## 기준 파일

- `data/settings/설정탭구조.csv`
- `data/settings/삭제키워드.csv`
- `data/settings/공사명_괄호정리.csv`

## 8대분류

1. 대쉬보드
2. 공고수집
3. 분류규칙
4. 공고체크 키워드
5. 마스터데이터
6. 검수
7. 내설정
8. 변경로그

## 적용 내용

- 검은 좌측 메뉴는 `설정탭구조.csv`의 `대분류`를 순서대로 읽어 만든다.
- 검은 좌측 메뉴 아이콘은 `설정탭구조.csv`의 `아이콘` 컬럼을 읽어 표시한다.
- 흰색 설정 패널은 선택된 `대분류` 안의 `소분류`와 `라벨`만 보여준다.
- 화면에 없는 기존 설정 탭은 `설정탭구조.csv`에서 제외하여 노출하지 않는다.
- 삭제키워드는 `id, 컬럼명, 검색키워드, 처리방법, 메모` 구조로 관리한다.
- 공사명 대괄호정리는 `id, 컬럼명, 검색키워드, 변경 후, 처리방법` 구조로 관리한다.
- 대쉬보드는 우측 상단 검수용 배지를 제거하고, `오늘 상태`와 `최근 변경`을 표시한다.
- 설정 저장 시 `data/settings/변경로그.csv`에 일시, 구분, 대상, 내용, 사용자, 메모를 기록한다.
- 마스터데이터는 모든 컬럼 전처리와 변경 판단의 기준 데이터다. 화면명과 컬럼 의미가 섞이지 않도록 탭 이름과 컬럼 이름을 문서와 UI에서 같이 관리한다.
- 모든 설정 탭에서 중간 회색 제목/설명 헤더를 제거해 표 영역을 넓힌다.
- 설정 상단은 검색, 다크모드, 닫기만 남긴다.
- 상단 breadcrumb, A1 로드 상태 배지, 중복 나가기 버튼은 제거한다.
- 다크모드 버튼은 설정 화면 전체에 실제 색상 테마를 적용한다.
- 공고수집 탭 순서는 `공고분류컬럼 설정`, `API 연결` 순서로 둔다.
- 공고수집의 컬럼 규칙 화면명은 `공고분류컬럼 설정`으로 쓴다.
- 공고수집의 API 관리 화면명은 `API 연결`로 쓴다.
- 표 상단 제목과 건수는 한 줄로 압축한다.
- 표 버튼명은 `CSV 다운` 대신 `내보내기`, `CSV 업로드` 대신 `가져오기`로 쓴다.
- 표에서 값을 바꾼 뒤 다른 설정 탭으로 이동하거나 설정을 닫으면 저장 여부를 먼저 확인한다.
- 표에서 값을 바꾼 뒤 브라우저 새로고침이나 닫기를 시도해도 저장하지 않은 변경을 경고한다.
- `발주처코드`처럼 500건을 넘는 표는 기본 500행 단위로 페이지 이동하고, 필요하면 1000행 또는 전체 표시로 바꾼다.
- 분류규칙의 2차 매핑 탭 이름은 `적격심사기준 2차변경`으로 쓴다.
- 특수실적에서 `PQ`는 발주처가 아니라 입찰방식이다. 예: `발주처=`, `입찰방식=PQ`.
- 종목매핑의 `상호진출_상대종목`은 별도 상단 편집 패널 없이 표 컬럼에서 직접 관리한다.
- `상호진출_상대종목` 값은 콤마로 구분한다. 예: `실내건축,금속창호지붕`.
- 설정 표의 `id`/`ID`는 내부 식별자 용도이므로 화면 컬럼에서는 숨긴다.
- 설정 표의 긴 내용은 2줄까지 보이되, 둘째 줄 하단이 잘리지 않도록 행 입력 높이를 확보한다.

## 표 공통 기능과 탭 전용 기능

표 공통 기능은 모든 설정 표에 동일하게 적용한다.

- 세로/가로 스크롤
- 열 너비 조정
- 헤더 클릭 정렬
- 2줄 표시와 hover 전체 보기
- 내보내기/가져오기
- 행 추가, 선택 삭제, 저장, 되돌리기
- 저장 전 이동 확인

탭 전용 기능은 해당 데이터의 의미가 맞을 때만 적용한다. 공고분류컬럼 설정의 `처리방법`, `우선순위`, `공고관리 표시`, `상세정보입력` 편집 방식은 다른 설정 탭에 퍼뜨리지 않는다. 예를 들어 삭제키워드의 `처리방법`은 삭제키워드 전용 값이므로 공고분류컬럼 설정의 드롭다운을 쓰지 않는다.

## 공고분류컬럼 설정

공고분류컬럼 설정은 서버공고 컬럼, 문서곡괭이 수집값, 계산맨 규칙을 최종 표준 컬럼으로 연결하는 기준표다.

| 컬럼 | 운영 의미 |
|---|---|
| 처리방법 | `계산`, `수집`, `둘다`, 공란 중 하나. 수집은 서버공고 또는 문서곡괭이 수집값이고, 계산은 참조방법 코드로 값을 만든다. |
| 우선순위 | `서버정보`, `문서곡괭이`, 공란 중 하나. 같은 컬럼에 두 출처가 있을 때 어떤 출처를 쓸지 정한다. |
| 참조방법 | 처리방법이 `계산` 또는 `둘다`일 때 사용하는 코드형 규칙이다. |
| 참조메모 | 참조방법 코드가 무엇을 계산하거나 조회하는지 사람이 읽을 수 있게 남긴 설명이다. |
| 컬럼입력목록 | 기존 `선택목록`의 화면 표시명. 해당 컬럼에 입력 가능한 목록이다. |
| 공고관리 표시 | 체크박스로 편집한다. 공고관리 표에서 필요한 컬럼만 보이게 조정하는 기준이다. |
| 상세정보입력 | 체크박스로 편집한다. 상세정보 입력 화면에서 필요한 컬럼만 보이게 조정하는 기준이다. |

`서버공고일치`, `확정메모`는 과거 서버공고 비교 검증과 컬럼 확정 과정에서 쓰던 메타다. 현재 운영 입력에는 핵심 컬럼이 아니므로 기본 화면에서는 숨기고, 데이터 파일에는 유지한다.

## 설정 화면 정리 원칙

설정 화면을 수정할 때는 먼저 요구사항을 문서로 정리한 뒤 작업한다.

예:

1. 제거 대상과 연결된 JSX, 상태값, import, CSS를 검색한다.
2. 단순 숨김 처리 대신 사용하지 않는 코드까지 정리한다.
3. 빌드와 브라우저 검수 후 문서를 갱신한다.
4. 의미 있는 작업 단위로 git commit/push 한다.

## 전처리 연결

- 삭제키워드:
  - 예: `컬럼명=공사명`, `검색키워드=[취]`, `처리방법=열삭제`
  - 전처리 시 해당 컬럼 값에 키워드가 포함되면 대상 row를 제외한다.
- 공사명 대괄호정리:
  - 예: `검색키워드=[종합심사낙찰제]`, `변경 후=[종심제]`
  - 전처리 시 해당 컬럼의 키워드를 변경 후 값으로 치환한다.

## 검수

- `npm run build` 통과.
- `node --check app/server.mjs` 통과.
- 브라우저에서 5174 설정 화면 확인:
  - 검은 메뉴 8개 노출.
  - 분류규칙 4개 탭 노출.
  - 마스터데이터 하위 묶음 노출.
  - 공고수집 탭 순서가 `공고분류컬럼 설정`, `API 연결` 순서로 노출.
  - 공고분류컬럼 설정의 표 상단 제목/건수가 한 줄로 노출.
  - `선택목록`은 화면에서 `컬럼입력목록`으로 노출.
  - `공고관리 표시`, `상세정보입력`은 체크박스로 편집.
  - `서버공고일치`, `확정메모`는 기본 화면에서 숨김.
  - 체크박스 수정 시 `저장 필요` 상태가 표시되고 되돌리기로 해제됨.
  - 공고분류컬럼 설정에서만 `처리방법`, `우선순위`이 드롭다운으로 표시됨.
  - 분류규칙 등 다른 설정 표에는 공고분류컬럼 설정 전용 드롭다운/체크박스가 적용되지 않음.
  - 우선순위 드롭다운은 공란, `서버정보`, `문서곡괭이`만 노출.
