# 미정사항

아직 결정하지 않은 항목만 남긴다.
답변이 끝난 항목은 결정기록과 각 룰 문서로 옮긴다.
질문할 때는 반드시 예시를 포함한다.
남은 항목은 구현 중 실제 케이스가 나올 때 예시와 함께 결정한다.

## 최근 답변 반영 완료

| 항목 | 반영 결과 |
|---|---|
| 표준 컬럼 목록 | 최종 엑셀에는 모든 표준 컬럼을 넣는다. 일부 대표 컬럼만 고르는 방식이 아니다. |
| 발주처 목록 | 발주처별 1차 개발대상 구분 질문은 삭제한다. 수집 기준은 서버공고 전체 컬럼이다. |
| 엑셀 출력 양식 | 기존 사용 중인 최종 엑셀 컬럼 순서와 양식을 가급적 맞춘다. |
| 공고문 파일 처리 | 첨부파일 자체를 직접 파싱하지 않는다. 기본은 API에서 받은 A3 HTML이다. |
| 적격발주처 중복 매칭 | 별도 우선순위가 아니라 설정된 순서대로 매칭한다. |
| 원발주처 특수기준 | 설정 마스터에서 관리한다. 예: `한국전력공사(배전)`, `한국전력공사(가공송전)`. |
| 적격심사평가기준 시행일 | 공고일 이전 기준 중 가장 최근 시행일의 적격기준을 사용한다. |
| 적격심사 제외 입찰방식 | 제외 대상이면 `적격평가기준_세부=빈값`이 정상이다. |
| 2차 매핑 빈 조건 | 빈 조건은 비교에서 빠진다. 예: `종목=빈값`, `공고확인=고난도`이면 공고확인만 본다. |
| 2차 매핑 충돌 | 조건 설정이 올바르면 동시에 충돌할 수 없다. 충돌하면 2차 세부조건 설정 오류다. |
| 1차/2차 키 불일치 예시 | 조달청 기준명은 `별표` 기준으로 본다. |
| 1차 기준표 관리 | CSV/JSON 가져오기뿐 아니라 웹 설정에서 행 추가/수정까지 가능해야 한다. |
| 적격 디버그 메타 | 새 이름은 `적격_1차상태`, `적격_1차사유`, `적격_2차상태`, `적격_2차사유`, `적격_처치방법`으로 한다. |
| 검색키워드 빈값 | 검색키워드가 없으면 파싱할 수 없다. 사람이 직접 선택·입력한다. |
| 특수실적 상위실적 | 계산맨도 쓰지 않는다. 사용자 검색용이다. |
| 특수실적_공통 적격업체 | 계산맨도 쓰지 않는다. 사용자 검색용이다. |
| 파서맨 조건판단형태 표시명 | 짧게 표시한다. 예: `3_1`, `1_2`. 필요하면 툴팁에서만 설명한다. |
| 파서맨 후보 저장 | 후보 테이블이나 별도 숨김 컬럼을 늘리지 않고 최종 대상 컬럼에 덮어쓴다. |
| 특수조건 입력 라우터 | `특수조건` 원문은 남긴다. 계산맨/라우터가 필요한 값을 `공고확인` 등 실제 참조 컬럼으로 복사한다. |
| 특수실적_공통 2차 매핑 | 2차 매핑표가 직접 `특수실적_공통`을 참조하지 않는다. 특수조건 입력값을 계산맨/라우터가 `공고확인` 같은 참조 컬럼으로 옮긴 뒤 그 컬럼을 본다. |
| 파서맨 UI 구조 | 사람이 확인하기 아주 편한 것이 최우선이다. 기존 iframe 유지 여부보다 검수 편의성을 기준으로 설계한다. |
| 검증 오류 표시 위치 | 특수실적 금액/면적/점수 누락 오류는 상세모달에서 빨갛게 표시한다. |
| 곡괭이전나라시 기능 범위 | 현재 작업흐름에 적은 범위를 우선 따른다. 추가 기능은 생길 때마다 논의한다. |
| 곡괭이전나라시 룰 화면 | 삭제키워드, 괄호정리, 적격발주처변경 등은 탭으로 나누는 방향이 관리하기 좋다. |
| 상호진출 매핑표 구조 | 종목매핑에 `상호진출_상대종목` 컬럼을 추가해 관리한다. |
| A2 첨부 fallback | 구현하지 않는다. 파싱은 A3 HTML만 사용한다. |
| 특수조건 라우터 실행 시점 | 입력 즉시가 아니라 저장 또는 2차분류 버튼 시점에 실행한다. |
| 파서맨 근거 표시 | 상세모달에 `rule_id`, `matched_keyword`, `source_text`를 보여준다. |
| 남은 미정 처리 방식 | 남은 항목은 지금 문답으로 모두 닫지 않고 구현 중 실제 케이스가 나올 때 결정한다. |

## 계속 열린 항목

| 항목 | 질문 예시 | 상태 |
|---|---|---|
| 종목별 적격심사기준 개별 호출 | 예: `토목=8억`, `전기=2억`이면 주공종 기준 A4만 볼지, 종목별 A4 호출이 필요한지 나중에 다시 판단한다. | 보류 |
| 송전사업소 데이터 | 예: `송전사업소_2026-05-24.csv`가 헤더만 있고 행이 없으면 데이터가 들어온 뒤 배전사업소와 같은 구조로 관리한다. | 데이터 대기 |
| 발전소지역 파일 | 예: 발전소지역 마스터가 첨부되면 폐광지역진흥지구처럼 특수지역 탭으로 둘지 확인한다. | 파일 대기 |
| 계산 컬럼 참조메모 | 예: `순공사_입찰점수`, `참여불가_시평액`은 사용자가 설명하면 참조메모와 참조방법 JSON으로 옮긴다. | 설명 받을 때마다 반영 |
