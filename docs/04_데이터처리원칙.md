# 데이터 처리 원칙

## 데이터 층 분리

원본값, 전처리값, 사람입력값, 최종값은 분리해서 관리한다.

| 데이터 층 | 설명 | 예시 |
|---|---|---|
| 원본값 | API나 파싱으로 받은 값 | `서울교육청` |
| 전처리값 | 코드와 룰로 표준화한 값 | `서울특별시교육청` |
| 사람입력값 | 사람이 직접 입력하거나 검증한 값 | `서울특별시교육청` |
| 최종값 | 출력에 사용할 값 | `서울특별시교육청` |

## 원본 보존

원본 데이터는 절대 덮어쓰지 않는다.

예시:

```text
원본 `dminsttNm=서울교육청`은 그대로 보관한다.
표준 컬럼 `발주처명`에만 `서울특별시교육청`을 넣는다.
```

## 빈값 보완

수집 데이터가 비어 있으면 해당 공고의 첨부 공고문을 불러와 파싱한다.

예시:

| 빈 컬럼 | 공고문 문구 | 후보값 |
|---|---|---|
| 지역제한 | 서울특별시 소재 업체 | 서울 |
| 업종 | 전기공사업 등록업체 | 전기공사업 |

## 보수적 선택

확실하지 않은 값은 채우지 않는다.

예시:

```text
공고문에 "수도권 업체 우대"라고만 있으면
`지역제한=서울`로 확정하지 않고 `검토필요`로 남긴다.
```

## 상태값

초기 상태값은 아래 범위로 제한한다.

```text
수집완료
전처리완료
검토필요
사람입력완료
최종확정
오류
```
