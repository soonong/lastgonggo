## 1차 답변 반영

### 표시 제어

`공고관리 표시`와 `상세정보입력`은 컬럼 표시 여부를 동적으로 제어하는 체크박스다.
입력 컬럼이 많으므로 사용자가 체크하면서 어떤 컬럼을 보여줄지 정한다.

예시:

```text
`공고관리 표시=true`이면 공고관리 표에 보이고,
`상세정보입력=true`이면 상세 입력 화면에 보인다.
```

### 설정 마스터

전처리 기준이 되는 파일들은 웹 설정 화면에서 추가, 변경, 수정할 수 있게 한다.
적격발주처 파일과 공고확인/특수실적 선택목록 파일은 첨부되면 구조를 확인해 설정 마스터로 반영한다.

예시:

```text
`한국가스공사 부산·경남지역본부 -> 한국가스공사` 같은 매핑은
코드가 아니라 설정 화면에서 관리한다.
```

첨부된 적격발주처 파일 구조:

```text
id, 발주처 키워드, 매핑 대상
```

예시:

```text
`한국남부발전 -> 조달청`
`한국동서발전 -> 조달청`
`한국전력공사 -> 한국전력공사`
```

확정 규칙:

```text
`발주처 키워드`는 포함검색으로 매칭한다.
`매핑 대상`은 `적격발주처`를 덮지 않고 `원발주처`에 넣는다.
```

예시:

```text
수집/홈페이지 기준이 `적격발주처=한국남부발전`이고
파일에 `한국남부발전 -> 조달청`이 있으면
`적격발주처`는 유지하고 `원발주처=조달청`으로 채운다.
```

### 컬럼 우선순위

문서곡괭이, LLM, 계산값 등 자동 후보값끼리의 선택은 컬럼 설명 CSV의 `우선순위`를 따른다.
사용자가 컬럼별로 직접 지정한다.
최종 목표는 문서곡괭이가 대부분의 값을 채우는 것이다.

예시:

```text
`A값`의 우선순위가 `LLM`이면 AI 추출값을 먼저 보고,
`입찰방식`의 우선순위가 `문서곡괭이`이면 문서곡괭이 추출값을 먼저 본다.
```

### 참조방법과 참조메모

설명이 있는 계산 컬럼부터 먼저 적용한다.
사용자가 컬럼을 설명하면, 그 내용을 `참조메모`에 사람이 읽을 수 있게 남기고, 규칙화 가능한 것은 `참조방법` JSON으로 옮긴다.

예시:

```text
사용자가 `순공사_입찰점수는 순공사원가 미달 감점이다`라고 설명하면
참조메모에 자연어 설명을 남기고, 계산식이 확정되면 참조방법 JSON으로 작성한다.
```

### 종목 개념

`종목`, `단독평가종목`, `종목세부JSON`은 한몸이다.
상세 규칙은 `docs/rules/종목룰.md`에 분리해서 관리한다.

예시:

```text
공고문 `전기공사업 등록업체`
  -> 표준 종목 `전기`
  -> 단독평가종목 `전기`
  -> 종목세부JSON에 `전기` 객체 생성
```

### 동일실적_평가여부

동일실적 컬럼에 평가비율이 있으면 `1`, 없으면 `0`이다.

예시:

```text
동일실적 평가비율이 있으면 `동일실적_평가여부=1`,
동일실적 평가비율이 없으면 `동일실적_평가여부=0`으로 둔다.
```

### 적격발주처, 원발주처, 적격평가기준_세부

적격심사기준을 찾기 위해 세 컬럼을 구분한다.

| 컬럼 | 의미 |
|---|---|
| 적격발주처 | 최초 수집 시 들어가며 수집 출처와 홈페이지 연결 기준으로 사용 |
| 원발주처 | 실제 평가에 사용할 적격심사기준 발주처 |
| 적격평가기준_세부 | A4 적격심사세부기준 API 호출용 세부 기준 |

예시:

```text
`적격발주처=한국전력공사`로 수집했지만
실제 기준이 배전특화공사이면 `원발주처=한국전력공사(배전)`으로 다시 잡고
그 다음 `적격평가기준_세부`를 찾는다.
```

### AI컬럼

`AI컬럼`은 AI가 채운 컬럼 목록을 기록하는 로그 성격이다.

예시:

```text
`AI컬럼=A값,상호진출_일반전문`이면
AI가 해당 컬럼들을 채웠다는 기록으로 본다.
```

### 예가_상한, 예가_하한

퍼센트 범위로 해석한다.

예시:

```text
`예가_상한=2.5`, `예가_하한=-2.5`이면
예정가격 변동 범위를 `+2.5%`, `-2.5%`로 본다.
```

### 공고확인, 공고확인_내용

`공고확인`은 사용자에게 확인 키워드를 보여주는 태그다.
`공고확인_내용`은 사용자가 텍스트를 입력해서 웹페이지에 보여주는 용도다.
파싱으로 입력 가능한 것은 자동 입력하고, 자동으로 못 하는 것은 사람이 입력한다.

예시:

```text
`공고확인=현장확인`이면 사용자에게 현장확인이 필요하다는 키워드를 보여준다.
`공고확인_내용=현장실사 후 계약상대자를 결정합니다`는 화면에 보여줄 설명 문장이다.
```

### 특수실적 계열

`특수실적`, `특수실적_공통`, `특수실적_내용`도 공고확인 계열과 같은 방식이다.
파싱으로 입력 가능한 것은 자동 입력하고, 자동으로 못 하는 것은 사람이 입력한다.

예시:

```text
`특수실적=철도신호_준공금액`은 확인 키워드 또는 분류값이고,
`특수실적_내용=최근 10년 이내 준공실적...`은 사용자가 확인할 원문 설명이다.
```

### 전문건설_주력분야

서버공고에서 받아온 값 또는 문서곡괭이가 `파싱용공고문`에서 가져온 값 중 선택해서 보여준다.
`단독평가종목` 규칙을 처리할 때 필요하다.

예시:

```text
서버공고의 `전문건설_주력분야=토공`이 있고,
파싱용공고문에서도 `토공`이 확인되면 이 값을 단독평가종목 계산에 참고한다.
```

### 공고번호 분리

`앞_공고번호`, `공고번호_차수`, `재입찰번호` 분리 규칙은 아래처럼 본다.

예시:

```text
`공고번호=R26BK01537599-000`이면
`앞_공고번호=R26BK01537599`, `공고번호_차수=0`, `재입찰번호=0`으로 본다.
```

### 참조방법과 참조메모 기반 컬럼

아래 컬럼은 `참조방법`과 `참조메모`를 토대로 만든다.

```text
순공사_입찰점수
참여불가_시평액
협정불가_시평액
경력기술자
공사성격
상호진출_일반전문
적격심사기준_열번호
```

예시:

```text
`순공사_입찰점수`는 임의 추정으로 채우지 않고,
CSV의 `참조방법` JSON과 `참조메모`에 있는 계산 규칙을 기준으로 만든다.
```

### 검증용 컬럼

검증용 컬럼은 현재 확정하지 않는다.
과거 수집 검증 때 썼던 컬럼이고, 검증 흐름은 다시 설계할 수 있다.

예시:

```text
`_v1_검증_상태`, `_v1_검증_메시지`는 지금 상태값을 정하지 않고 보류한다.
```

## 남은 질문

### Q1. 공고확인/특수실적 선택목록 파일을 첨부하면 어떤 값이 화면 표시명이고 어떤 값이 내부 코드인가요?

예시:

```text
파일에 `코드=SITE_CHECK`, `표시명=현장확인`, `설명=현장 확인 필요`가 있으면
화면에는 `현장확인`을 보여주고 내부 저장은 `SITE_CHECK`로 할까요?
```

### Q2. 종목 묶기 규칙을 알려줄까요?

예시:

```text
`건축 + 토목 -> 토건`,
`금속구조물 + 지붕판금 -> 금속창호지붕`,
`건축 + 실내건축`은 상호진출이라 둘 다 유지
같은 묶기 규칙을 알려주면 `docs/rules/종목룰.md`에 바로 추가하겠습니다.
```

### Q3. 적격발주처 키워드가 여러 개 동시에 매칭되면 무엇을 우선할까요?

예시:

```text
원본 문자열에 `한국전력공사`와 더 긴 키워드가 동시에 걸리면
더 긴 키워드를 우선할까요, 아니면 설정 파일의 `id`가 작은 행을 우선할까요?
```
