# 공고확인·특수실적·특수실적_공통 답변 반영

작성일: 2026-05-26

## 목적

`공고확인`, `특수실적`, `특수실적_공통` 설정 데이터에 대한 사용자 답변을 반영한다.
이 문서는 기존 초안의 미정사항을 줄이고, 문서곡괭이·계산맨·2차 매핑과 연결할 때 헷갈리기 쉬운 부분을 고정한다.

핵심 원칙:

```text
세 파일은 합치지 않는다.
종목/발주처/대분류는 결과값을 찾기 위한 화면 검색·필터용 메타다.
실제 컬럼에 들어가는 값은 결과값이다.
```

예시:

```text
특수실적 탭에서
종목=전기
발주처=한국철도공사
대분류=철도신호
결과값=철도신호설비_준공금액

위 행의 종목/발주처/대분류는 사용자가 결과값을 찾기 위한 분류다.
공고 row의 종목이나 발주처와 자동 매칭하는 조건으로 쓰지 않는다.
```

## 공통 확정 규칙

| 항목 | 확정 규칙 | 예시 |
|---|---|---|
| 결과값 중복 | 같은 탭 안에서 `결과값`은 오직 1개여야 한다 | 공고확인 탭에 `전자시담`이 2개 있으면 DB 오류 |
| 여러 결과 | 한 공고에서 여러 값이 나오면 `/`로 연결한다 | `공고확인=전자시담/기초금액발표전` |
| 결과 순서 | 순서는 중요하지 않다 | `A/B`와 `B/A`는 업무상 같은 후보 |
| 검색키워드 여러 줄 | OR 조건이다 | `전자시담`, `전자 시담` 중 하나만 맞아도 매칭 |
| 와일드카드 | 검색키워드에서 `*` 사용 가능 | `대표사 포함 *개 업체` |
| 제외키워드 | 검색키워드가 매칭된 문맥 안에서만 비교한다 | 같은 문단의 `허용하지 않음`은 제외, 먼 문단은 제외 아님 |
| 검색키워드 빈값 | 자동 파싱 대상이 아니라 사람이 직접 선택·입력한다 | `공연장[3000M]_준공금액` 키워드 빈값이면 사람 선택 |
| 메모 | 사람 설명용이다 | 로직에서 읽지 않음 |
| id | 가져오기 때 새로 발급한다 | CSV의 id를 그대로 신뢰하지 않음 |

## 컬럼명 정정

기존 CSV의 일부 컬럼명은 업무 의미와 맞지 않는다.
새 구조에서는 화면 표시명과 내부 의미를 분리한다.

| 기존 컬럼 | 새 해석 | 예시 |
|---|---|---|
| 공고확인 파일의 `종목` | 잘못된 옛 컬럼명, 실질적으로 고정 구분값 | 값이 `공고확인`이어도 실제 종목 아님 |
| 특수실적_공통 파일의 `종목` | 이동컬럼명 또는 구분값 | 값이 `특수실적_공통`이면 해당 컬럼으로 이동 |
| `발주처` | 결과값 검색용 메타 | `행자부`, `한국철도공사` |
| `대분류` | 화면 필터 | `심사방법`, `철도신호`, `기업인증` |
| `상위실적` | 사용자 검색용 메타. 문서곡괭이와 계산맨에서 사용하지 않음 | 검색 화면에서 관련 결과값 찾기 |

예시:

```text
공고확인 파일:
종목=공고확인
발주처=행자부
대분류=심사방법
결과값=전자시담

위 값들은 "행자부 공고일 때만 자동 적용"이 아니라
사용자가 공고확인 결과값을 찾기 위한 분류다.
```

## 특수조건 입력 라우터

사람이 `특수조건` 같은 입력란에 값을 넣으면,
시스템은 세 설정 탭에서 해당 키워드나 결과값을 찾아 실제 목적 컬럼으로 옮긴다.

흐름:

```text
사람 입력 특수조건
  -> 공고확인 설정 검색
  -> 특수실적 설정 검색
  -> 특수실적_공통 설정 검색
  -> 일치한 결과값을 목적 컬럼에 복사
```

예시:

```text
사람 입력:
특수조건=기초금액발표전

공고확인 설정:
결과값=기초금액발표전
검색키워드=기초금액 발표 전

결과:
공고확인=기초금액발표전
```

예시:

```text
사람 입력:
특수조건=사회적기업

특수실적_공통 설정:
결과값=사회적기업
검색키워드=사회적기업

결과:
특수실적_공통=사회적기업
```

주의:

```text
특수조건은 사람이 자유롭게 넣는 입구다.
최종 계산과 2차 매핑은 `공고확인`, `특수실적` 같은 실제 참조 컬럼을 본다.
`특수조건` 원문은 남긴다.
```

## 공고확인

공고확인은 사용자가 확인해야 하는 태그 또는 세부 분류 조건이다.

확정 규칙:

```text
공고확인_내용은 사람이 채운다.
공고확인은 2차 매핑 조건으로 들어갈 수 있다.
2차 매핑에서 공고확인은 전체 값이 정확히 일치해야 한다.
```

예시:

```text
row.공고확인=배전공가/전자시담
2차 매핑 조건 공고확인=배전공가
  -> 불일치

2차 매핑 조건 공고확인=배전공가/전자시담
  -> 일치
```

## 특수실적

특수실적은 종목에 대한 실적 조건이다.
자동으로 금액·면적·점수까지 확정하지 않고, 우선 텍스트 결과값만 넣는다.

확정 규칙:

```text
특수실적은 2차 매핑 전에 완료되어야 한다.
2차 매핑에서 특수실적은 포함검색으로 본다.
특수실적 값은 일단 텍스트만 저장한다.
금액/면적/점수는 사람이 입력한다.
```

예시:

```text
row.특수실적=무정전_준공금액/철도신호설비_준공금액
2차 매핑 조건 특수실적=무정전_준공금액
  -> 포함되므로 일치
```

특수실적 결과값의 대괄호는 같은 카테고리 안의 세부분류다.

예시:

```text
철도배전선로[154Kv]_준공금액
  -> 철도배전선로 카테고리 안의 154Kv 세부조건
```

`_준공금액`, `_준공면적`, `_준공점수` 같은 접미사는 값의 성격을 알려준다.
하지만 현재 자동 단위 변환은 하지 않는다.

예시:

```text
특수실적=공연장[3000M]_준공면적
면적 숫자 없음
  -> 빨간 오류 또는 검토필요로 표시하고 사람이 입력
```

## 특수실적_공통

특수실적_공통은 종목 실적이 아니라 사업자 조건이다.
기업인증, 신용평가, 장비, 협력업체 같은 조건이 여기에 들어간다.

확정 규칙:

```text
특수실적과 특수실적_공통은 다른 컬럼이다.
특수실적은 종목에 대한 내용이다.
특수실적_공통은 사업자에 대한 조건이다.
특수실적_공통은 2차 매핑표가 직접 참조하지 않는다.
```

예시:

```text
특수실적=무정전_준공금액
  -> 종목 실적 조건

특수실적_공통=사회적기업
  -> 사업자 조건
```

`적격업체` 컬럼은 사용자 검색용 메타다.
문서곡괭이와 계산맨이 직접 사용하지 않는다.

예시:

```text
특수실적_공통 결과값=사회적기업
적격업체=true
  -> 문서곡괭이는 특수실적_공통=사회적기업만 채움
  -> 적격업체=true는 사용자가 설정 화면에서 찾기 위한 메타
```

## 문서곡괭이 연결

문서곡괭이는 컬럼별 설정 조건을 한 번에 실행한다.
`공고확인`, `특수실적`, `특수실적_공통`도 각각 하나의 대상 컬럼으로 실행한다.

예시:

```text
본문:
본 공사는 전자시담으로 진행하며 사회적기업 확인서를 제출해야 합니다.

공고확인 설정:
검색키워드=전자시담
결과값=전자시담

특수실적_공통 설정:
검색키워드=사회적기업
결과값=사회적기업

결과:
공고확인=전자시담
특수실적_공통=사회적기업
```

설정 목록에 없는 값은 반환하지 않는다.

예시:

```text
본문에 `신규특수조건ABC`가 있음
세 설정 탭 어디에도 결과값 또는 검색키워드가 없음
  -> 빈값 반환
```

## 재전처리 연결

사람이 `공고확인`, `특수실적`, `특수실적_공통`을 수정하면
문서곡괭이를 다시 돌리지 않고 필요한 후속 처리만 실행한다.

확정된 방향:

```text
공고확인 변경 -> 2차 매핑 재실행
특수실적 변경 -> 2차 매핑 재실행
특수실적_공통 변경 -> 직접 2차 매핑하지 않음. 특수조건 라우터가 공고확인 등으로 옮긴 값이 있으면 그 컬럼 기준으로 후속 처리
사람 입력값은 문서곡괭이으로 다시 덮지 않음
```

예시:

```text
사람이 공고확인=배전공가로 수정
  -> 공고확인 값은 보호
  -> 적격심사기준 2차 매핑 재실행
  -> 원발주처/적격평가기준_세부가 바뀔 수 있음
```

## 검증 기준

| 검증 | 처리 | 예시 |
|---|---|---|
| 결과값 중복 | 오류 | 같은 탭에 `전자시담` 2개 |
| 검색키워드 빈값 | 오류 아님 | 사람이 직접 선택할 값 |
| 설정에 없는 파서 결과 | 빈값 | 신규 문구는 자동 채택 안 함 |
| 특수실적 금액/면적/점수 누락 | 빨간 오류 | `준공금액` 조건인데 금액 없음 |
| 2차 매핑 조건값 없음 | 조용히 통과/미매칭 | 조건값이 설정에 없어도 즉시 오류는 아님 |

## 아직 질문이 남은 지점

아래는 다음 문서의 질문 목록에서 다시 확인한다.

```text
1. 특수조건 입력값을 원문 그대로 남길지, 목적 컬럼으로 이동 후 비울지
2. 결과값 중복 검증을 탭 안에서만 할지, 세 탭 전체에서도 경고할지
3. 특수실적 금액/면적/점수 누락을 상세모달에서 어떻게 보여줄지
```
