# 탭별 정의 목록과 질문지 (2026-05-30)

이 문서는 기능을 새로 추가하거나 바꿀 때 먼저 확인할 기준 문서다. 각 탭은 역할이 섞이면 안 되며, 버튼과 데이터 이동도 해당 탭 책임 안에서만 처리한다.

## 공통 원칙

- 작업 전 관련 문서를 먼저 확인한다.
- 기존 정의와 충돌하면 바로 구현하지 않고 예시를 들어 다시 질문한다.
- 표의 행 선택 기능은 해당 탭에서 명시된 버튼에만 적용한다.
- 설정/마스터 데이터는 가능한 한 하드코딩하지 않는다.
- 문서 파싱 관련 명칭은 `문서곡괭이`로 통일한다.

## 입찰공고 원본

목적: A1 서버공고 API에서 원본 공고 row를 가져온다.

버튼:

- API 호출: `자동수집여부=1`인 공고만 가져온다. 입력일이 있으면 해당 날짜 포함 이후 공고만 가져온다. 종목 필터가 있으면 API 수집 조건으로 적용한다.
- 곡괭이전나라시: 서버공고 raw를 삭제키워드, 괄호정리, 기본 정규화 기준으로 전처리해서 `입찰공고 전처리`로 넘긴다.
- 모두 삭제: 현재 원본/후속 단계 데이터를 초기화한다.

질문 필요:

- API 호출 실패 row를 원본 탭에 남길지, 로그에만 남길지 예시 필요.
- 곡괭이전나라시에서 추가될 룰은 어떤 설정 탭에 붙일지 매번 확인한다.

## 입찰공고 전처리

목적: 곡괭이전나라시가 끝난 데이터를 확인하고, A3 문서곡괭이 실행 전 상태를 점검한다.

버튼:

- 문서곡괭이질 시작: A3 파싱용공고문을 불러와 설정된 문서곡괭이 룰로 빈값과 검토값을 채운다.
- 테스트 내보내기/가져오기: 문서곡괭이 재실행 시간이 길 때 검증용 CSV를 저장/복원한다.

질문 필요:

- 문서곡괭이 실패 공고는 공고관리로 보낼지, 전처리에 남겨 재시도할지 예시 필요.

## 공고관리

목적: 문서곡괭이 이후 사람이 종목 단위로 검수하고 필요한 값만 직접 입력한다.

종목 목록 규칙:

- 종목은 `종목` 컬럼을 `/`, `,`, 괄호 기준으로 토큰화한 뒤 정확히 일치하는 토큰만 센다.
- `건축`은 `실내건축`과 다르다. `조경`은 `조경식재`와 다르다.
- 2차분류 완료로 이동한 공고번호는 공고관리의 모든 종목 카운트에서 빠진다.
- 완료 표시는 `2차분류_종목`에 기록된 선택 종목만 `완료`로 표시한다.
- 예: `건축/조경` 공고 1건을 `건축`으로 완료하면 `건축 완료`만 붙고, `조경` 카운트에서는 해당 공고가 빠진다.

버튼:

- 삭제: 체크한 공고만 공고관리 대상에서 삭제한다.
- 재전처리: 체크한 공고만 사람 입력값 기준으로 필요한 컬럼을 다시 계산한다.
- 2차분류: 체크한 공고만 곡괭이후나라시/2차분류 처리 후 `2차분류 완료`로 이동한다.

상세모달:

- 공사명 더블클릭으로 연다.
- 선택 공고가 상단 행 스트립 첫 줄에 온다.
- 좌측은 공고문/첨부파일 미리보기, 우측은 입력 필드 탭이다.
- 공고문 검색은 원문 HTML 구조를 유지한 채 하이라이트만 추가한다.

질문 필요:

- 완료한 공고를 다시 공고관리로 되돌리는 버튼이 필요한지.
- 종목 미확인 row를 어떤 조건에서 완료 처리할지.
- 상세모달 저장 후 자동으로 다음 공고로 이동할지, 닫기만 할지.

## 2차분류 완료

목적: 공고관리에서 완료 처리한 row만 모아 최종 확인한다.

버튼:

- 삭제: 체크한 완료 row를 2차분류 완료 목록에서 삭제한다.
- 엑셀 추출: 완료 row를 최종 파일로 출력한다.

질문 필요:

- 2차분류 완료에서 삭제한 row를 공고관리로 되돌릴지, 완전 삭제로 볼지.
- 완료 row의 `2차분류_종목`을 사람이 수정할 수 있어야 하는지.

## 상세모달 좌측 패널

목적: 공고문과 첨부파일을 보면서 우측 필드를 입력한다.

구성:

- 기본 탭은 `공고문`.
- 첨부파일 탭에서 파일을 클릭하면 좌측에 파일별 탭이 추가된다.
- 검색창은 공고문 탭과 같은 줄 우측에 둔다.
- Ctrl+F/Cmd+F, Enter, Shift+Enter, Escape를 지원한다.

질문 필요:

- PDF 첨부파일은 브라우저 PDF 검색에 맡길지, 텍스트 변환까지 할지.
- 첨부파일이 여러 개일 때 파일명 정렬 기준.

## 상세모달 우측 패널

탭 순서:

```text
추가 -> 종목 -> 기본 -> 기타 -> 첨부파일 -> 적격심사기준
```

탭별 질문 필요:

- 추가: `공고확인`, `특수실적`, `특수실적_공통`, `공사현장`의 필수 입력 기준.
- 종목: 단독평가종목 직접 수정 후 어느 컬럼까지 재계산할지.
- 기본: 발주처/지역/입찰방식 같은 선택목록 검증 실패 표시 방식.
- 기타: 계산 컬럼을 사람이 덮어쓸 수 있는지.
- 첨부파일: A2 파일 목록 API 결과를 어디까지 캐시할지.
- 적격심사기준: A4 결과가 없을 때 오류인지 검토필요인지.

## 설정/룰관리

목적: 모든 전처리 기준과 마스터 데이터를 관리한다.

주의:

- CSV 가져오기/내보내기는 최신 설정에 즉시 반영되어야 한다.
- 저장하지 않은 변경점이 있으면 다른 탭 이동 전 저장 여부를 물어야 한다.
- 컬럼명 변경은 문서와 코드 참조명을 함께 검색해서 맞춘다.
