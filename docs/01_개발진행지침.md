# 개발 진행 지침

## 개발 전 확인 순서

모든 개발 작업은 아래 순서로 진행한다.

```text
1. 관련 문서 확인
2. 기존 원칙과 충돌 여부 확인
3. 충돌이 있으면 먼저 설명하고 정리
4. 충돌이 없으면 개발 진행
5. 개발 후 검증
6. 문서 업데이트 필요 여부 확인
```

## 질문 규칙

질문은 반드시 예시를 들어서 한다.

좋은 질문 예시:

```text
사람이 `공고명=학교 급식실 개선공사`로 수정하면
`공고분류`, `종목그룹`, `키워드`만 다시 계산하면 될까요?
아니면 `지역제한`까지 다시 봐야 할까요?
```

나쁜 질문 예시:

```text
어떤 컬럼을 다시 처리할까요?
```

## 충돌 처리

기존 문서와 새 요청이 충돌하면 바로 개발하지 않는다.

예시:

```text
새 요청: 사람이 입력한 값을 자동 전처리값으로 덮어써도 된다.
기존 원칙: 사람 입력값은 최우선이며 코드나 AI가 덮어쓰면 안 된다.
처리: 충돌을 먼저 설명하고 기준을 다시 확정한 뒤 개발한다.
```

## 개발 중 기록해야 할 것

| 항목 | 설명 | 예시 |
|---|---|---|
| 처리 방법 | 어떤 방식으로 값을 만들었는지 | 컬럼 매핑, 공고문 파싱, 코드 계산 |
| 선택 이유 | 왜 그 값을 선택했는지 | 사람 검증값이라 최우선 적용 |
| 출처 | 값이 어디서 왔는지 | API, 공고문, 사람 입력, AI 후보 |
| 상태 | 최종 사용 가능 여부 | 확정, 검토필요, 오류, 빈값 |
