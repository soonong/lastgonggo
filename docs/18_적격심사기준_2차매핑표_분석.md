## 파일 성격

이 파일은 1차 적격심사평가기준 결과를 받은 뒤,
A~H 조건 영역을 보고,

L 이후 컬럼은 계산하거나 추가로 덮어쓸 수 있는 확장 컬럼이다.

## 확인된 특징

### 결과값 형식은 일관된다

404행 모두 `적격평가기준_세부`가 `원발주처_기준명` 형태로 시작한다.

예시:

```text
원발주처=한국전력공사(배전)
적격평가기준_세부=한국전력공사(배전)_3억미만
```

### 조건 컬럼은 B~H다

기존 초안처럼 `공고확인` 하나만 보는 구조로는 부족하다.
실제 2차 매핑표는 아래 조건을 함께 본다.

```text
적격평가기준_세부 (매칭키)
입찰방식
종목
공고확인
특수실적
등급공사
동일실적평가여부
```

예시:

```text
매칭키=한국전력공사_50억미만
입찰방식=적격심사
종목=전기
특수실적=무정전_준공금액,배전사선_준공금액,...
동일실적평가여부=1
  -> 한국전력공사(배전)_50억미만
```

`지역제한`, `추정가격기준`, `추정금액기준`, `기초금액기준`은 조건이 아니라
매칭 후 계산하거나 덮어쓸 수 있는 L 이후 확장 컬럼이다.

### 빈 조건은 전체 통과로 해석할 가능성이 높다

예시:

```text
종목=빈값
공고확인=고난도
  -> 종목과 무관하게 고난도 조건만 맞으면 후보
```

다만 이 해석은 사용자 확인이 필요하다.

조건 설정이 올바르면 두 결과가 동시에 걸릴 수 없다.

## 구조 반영 방향

2차 매핑표는 고정된 `공고확인` 전용 테이블이 아니라
여러 조건 컬럼을 가진 조건 매칭 테이블로 둔다.

권장 구조:

| 구분 | 컬럼 |
|---|---|
| 기본키 | `id` |
| 입력키 | `적격평가기준_세부 (매칭키)` |
| 조건 | `적격평가기준_세부 (매칭키)`, `입찰방식`, `종목`, `공고확인`, `특수실적`, `등급공사`, `동일실적평가여부` |
| 핵심 덮어쓰기 | `원발주처`, `적격발주처`, `적격평가기준_세부` |
| 추가 덮어쓰기 | `지역제한`, `추정가격기준`, `추정금액기준`, `기초금액기준`, 이후 추가 컬럼 |
| 관리 | `사용여부`, `메모`, `편집` |

현재 파일에는 `사용여부`가 없으므로 웹 설정에 올릴 때 추가하는 편이 안전하다.
동시 충돌은 우선순위로 해결하지 않고 조건세부 설정 오류로 고친다.

## 확인 질문

1. 빈 조건은 전체 통과인가?

예시:

```text
종목=빈값
공고확인=고난도
```

이 경우 종목은 아무거나 통과하고 `공고확인=고난도`만 보면 되는지 확인이 필요하다.

2. 충돌 행은 어떤 조건으로 구분해야 하나?

예시:

```text
조달청_별표-1-5 + 간이종심제
  -> 조달청(간이종심제)
  -> 한국철도공사(간이종심제)
```

두 결과가 동시에 걸리면 조건세부 설정이 잘못된 것이다.
우선순위로 고르지 않고 조건을 수정한다.

3. 1차 기준표에 없는 매칭키는 1차 기준표를 보강해야 하나?

예시:

```text
2차 매칭키=조달청_별표-3
현재 1차 기준표에는 없음
```

이 경우 1차 기준표에 `조달청_별표-3`이 나오도록 보강해야 하는지,
아니면 2차 매칭키는 다른 방식으로 생성되는지 확인이 필요하다.
