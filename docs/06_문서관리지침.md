# 문서 관리 지침

## 문서 분리 기준

문서는 길어지지 않게 역할별로 나눈다.

| 위치 | 역할 |
|---|---|
| `00_프로젝트개요.md` | 바뀌면 안 되는 큰 원칙 |
| `01_개발진행지침.md` | 개발 전에 확인할 절차 |
| `02_작업흐름.md` | 수집부터 출력까지의 흐름 |
| `03_페이지구성.md` | 화면과 메뉴 기준 |
| `04_데이터처리원칙.md` | 원본, 전처리, 사람입력, 최종값 기준 |
| `05_사람입력_재전처리.md` | 사람 입력 후 부분 재전처리 기준 |
| `07_검증기준.md` | 최종 출력 전 검사 기준 |
| `08_결정기록.md` | 왜 그렇게 정했는지 남기는 기록 |
| `09_미정사항.md` | 아직 정하지 않은 질문 |
| `rules/` | 실제 처리 룰 |
| `examples/` | 케이스별 예시 |

## 문서 길이 원칙

| 문서 종류 | 길이 기준 | 작성 방식 |
|---|---|---|
| 개요 문서 | 짧게 유지 | 원칙만 기록 |
| 룰 문서 | 표 중심 | 처리방법, 이유, 예시 |
| 예시 문서 | 케이스별 분리 | 한 상황을 자세히 설명 |
| 결정기록 | 날짜별 누적 | 결정, 이유, 예시 |

## 문서 업데이트 기준

아래 상황에서는 문서를 업데이트한다.

| 상황 | 업데이트 위치 | 예시 |
|---|---|---|
| 새 원칙이 생김 | 관련 상위 문서 | 사람 입력값은 절대 자동 덮어쓰기 금지 |
| 새 룰이 생김 | `rules/` | `공고명` 변경 시 `키워드` 재처리 |
| 실제 예외 발견 | `examples/` | 나라장터 시설에서 업종이 `기타`로 들어오는 사례 |
| 중요한 판단 변경 | `08_결정기록.md` | 페이지를 5개 메뉴로 제한하기로 결정 |
