# 사업소·특수지역 파일 분석

## 목적

공사현장에는 일반 행정구역뿐 아니라 배전사업소, 송전사업소, 폐광지역진흥지구 같은 특수 지역값이 들어올 수 있다.
이 값들은 일반 `지역디비.xlsx`와 성격이 다르므로 별도 마스터로 관리한다.

## 분석 파일

| 파일 | 행 수 | 컬럼 | 비고 |
|---|---:|---|---|
| `송전사업소_2026-05-24.csv` | 0 | `id`, `지역본부`, `사업소명`, `인접 사업소` | 현재 헤더만 있음 |
| `한전_배전사업소_2026-05-24.csv` | 197 | `id`, `지역본부`, `사업소명`, `인접 사업소` | 실제 사업소 데이터 있음 |
| `폐광지역진흥지구_2026-05-24.csv` | 7 | `id`, `광역`, `시·군`, `상세 구역` | 특수 구역 데이터 있음 |

## 배전사업소

파일 구조:

| 컬럼 | 의미 | 예시 |
|---|---|---|
| id | 정렬/식별값 | `1` |
| 지역본부 | 한전 지역본부 | `서울` |
| 사업소명 | 공사현장에 표시할 사업소명 | `서울본부직할` |
| 인접 사업소 | 인접 사업소 목록 | `서대문은평지사, 마포용산지사` |

지역본부별 행 수:

| 지역본부 | 행 수 |
|---|---:|
| 서울 | 14 |
| 인천 | 10 |
| 경기 | 27 |
| 강원 | 17 |
| 충북 | 11 |
| 대전세종충남 | 18 |
| 전북 | 14 |
| 광주전남 | 27 |
| 대구 | 16 |
| 경북 | 12 |
| 부산울산 | 13 |
| 경남 | 18 |

예시:

```text
지역본부=서울
사업소명=강남지사
인접 사업소=서초지사, 강동송파지사
```

## 송전사업소

파일 구조는 배전사업소와 같다.
현재는 헤더만 있고 데이터 행은 없다.

예시 구조:

```text
id,지역본부,사업소명,인접 사업소
```

주의:

```text
송전사업소 데이터가 들어오면 배전사업소와 같은 로직으로 관리한다.
현재는 빈 마스터로 등록해도 된다.
```

## 폐광지역진흥지구

파일 구조:

| 컬럼 | 의미 | 예시 |
|---|---|---|
| id | 정렬/식별값 | `1` |
| 광역 | 광역 표준값 | `강원` |
| 시·군 | 시군구 표준값 | `태백시` |
| 상세 구역 | 읍면동 또는 구역 설명 | `시 전역` |

광역별 행 수:

| 광역 | 행 수 |
|---|---:|
| 강원 | 4 |
| 경북 | 1 |
| 충남 | 1 |
| 전남 | 1 |

예시:

```text
광역=강원
시·군=삼척시
상세 구역=도계읍
```

예시:

```text
광역=충남
시·군=보령시
상세 구역=대천동, 웅천읍, 청라면, 남포면, 주산면, 성주면, 미산면
```

## 관리 방향

사업소·특수지역은 일반 행정구역과 분리해 관리한다.
단, 사용 화면은 같은 `지역설정` 안에 둔다.

권장 탭 구조:

```text
지역설정
  - 행정지역
  - 배전사업소
  - 송전사업소
  - 폐광지역진흥지구
```

이유:

```text
행정지역은 지역제한과 검색용현장의 표준 주소지 매핑에 쓰인다.
배전/송전사업소와 폐광지역진흥지구는 공사현장 또는 지역평가점수 계산에 쓰이는 특수 기준이다.
같은 지역설정 안에서 관리하되, 서로 다른 테이블로 둬야 컬럼 의미가 섞이지 않는다.
```
