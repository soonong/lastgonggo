# 지역매핑 룰

## 목적

지역매핑은 수집값 또는 파싱값으로 들어온 지역 표현을 표준 지역값으로 바꾸는 전처리 규칙이다.

대상 컬럼:

```text
지역제한
공사현장
검색용현장
```

`검색용현장`은 `지역제한`과 같은 방식으로 처리한다.
별도 설명은 두지 않는다.

## 기준 파일

지역 기준은 `지역디비.xlsx`를 사용한다.
이 파일은 웹 설정의 지역 설정 탭에서 추가, 변경, 수정할 수 있어야 한다.

핵심 컬럼:

| 컬럼 | 의미 | 예시 |
|---|---|---|
| 광역시/도-전처리전 | 수집/파싱값과 비교할 광역 alias | `경기도/경기` |
| 광역시/도-전처리후 | 최종 광역 표준값 | `경기` |
| 시군구-전처리전 | 수집/파싱값과 비교할 시군구 alias | `구리시` |
| 시군구-전처리후 | 최종 시군구 표준값 | `구리시` |
| 하위구-전처리전 | 수집/파싱값과 비교할 하위구 alias | `분당구/분당구` |
| 하위구-전처리후 | 최종 하위구 표준값 | `분당구` |
| 시군구-홈페이지용 | 짧은 시군구 alias | `구리` |
| 인접지역 | 인접지역 참조 후보 | `남양주시` |

예시:

```text
수집값=경기도
광역시/도-전처리전=경기도/경기
광역시/도-전처리후=경기
  -> 지역제한=경기
```

## 지역제한 기본 구조

지역제한은 주소지 기준으로 처리한다.

우선 범위:

```text
1. 전국
2. 광역시/도
3. 시군구
4. 하위구/동읍
```

`전국`은 모든 지역을 뜻한다.
지역디비 파일에 없더라도 내장 특수값으로 처리한다.
다른 지역과 함께 들어온 경우에는 `전국`으로 축약하지 않고 `/` 구분 형식을 유지한다.

예시:

```text
수집값=전국
  -> 지역제한=전국

수집값=전국/울산
  -> 지역제한=전국/울산
```

## 표시 형식

광역시/도만 있으면 광역시/도 표준값만 표시한다.
시군구가 있으면 `광역시/도,시군구` 형식으로 표시한다.
여러 지역은 `/`로 구분한다.

예시:

```text
수집값=경기도
  -> 광역시/도 매핑
  -> 지역제한=경기
```

예시:

```text
수집값=구리
  -> 시군구-홈페이지용 또는 시군구 매핑
  -> 시군구=구리시
  -> 광역=경기
  -> 지역제한=경기,구리시
```

예시:

```text
수집값=서울/경기도/강원
  -> 서울/경기/강원
```

예시:

```text
수집값=남양주/구리
  -> 남양주시/구리시
  -> 지역제한=경기,남양주시/경기,구리시
```

## 전처리 절차

수집값 또는 파싱값이 들어오면 먼저 `/`로 나누어 각각 매핑한다.
각 token 안에 광역과 시군구가 함께 들어오면 다시 쪼개어 각각 전처리한 뒤 합친다.
공고문 자체가 지역을 잘못 적는 경우가 있으므로, 합친 결과가 지역디비의 상하관계와 맞는지 확인한다.

```text
1. 원본 지역 문자열을 `/`로 분리한다.
2. 각 token의 앞뒤 공백을 제거한다.
3. token이 `전국`이면 `전국`으로 확정한다.
4. token 안에 `경기도 구리시`처럼 여러 지역 단어가 있으면 단어별로 쪼개어 매핑한다.
5. 광역시/도-전처리전 alias에서 token 또는 쪼갠 단어를 찾는다.
6. 없으면 시군구-전처리전 alias에서 찾는다.
7. 없으면 시군구-홈페이지용 alias에서 찾는다.
8. 없으면 하위구-전처리전 alias에서 찾는다.
9. 광역과 시군구가 함께 매칭되면 지역디비상 같은 상하관계인지 확인한 뒤 합친다.
10. 매칭 결과가 1개면 전처리후값으로 지역제한을 만든다.
11. 매칭 결과가 여러 개면 자동 확정하지 않고 검토필요로 둔다.
12. 매칭 결과가 없으면 빈값 또는 검토필요로 둔다.
```

예시:

```text
수집값=구리
  -> 시군구-홈페이지용=구리 1건 매칭
  -> 시군구-전처리후=구리시
  -> 광역시/도-전처리후=경기
  -> 지역제한=경기,구리시
```

예시:

```text
수집값=경기도 구리시
  -> 경기도=경기
  -> 구리시=경기,구리시
  -> 광역과 시군구 상하관계 일치
  -> 지역제한=경기,구리시
```

예시:

```text
수집값=중구
  -> 서울 중구, 부산 중구, 대구 중구, 인천 중구, 대전 중구, 울산 중구가 동시에 매칭
  -> 자동 확정하지 않음
  -> 사람 입력 또는 공고문 근거 확인
```

예시:

```text
수집값=부산 중구
  -> 부산=부산
  -> 중구 후보 중 부산 중구로 좁혀짐
  -> 지역제한=부산,중구
```

## 보수적 선택

지역명은 같은 이름이 여러 광역에 존재할 수 있다.
확실하지 않은 값은 억지로 채우지 않는다.

예시:

```text
수집값=고성군
  -> 강원,고성군 / 경남,고성군 모두 가능
  -> 자동 확정하지 않음
```

예시:

```text
수집값=경기 고성군
  -> 경기 광역과 고성군 후보가 상하관계로 맞지 않음
  -> 공고문 오기 가능성
  -> 자동 확정하지 않고 검토필요
```

예시:

```text
수집값=수도권
  -> 서울/경기/인천으로 임의 확장하지 않음
  -> 검토필요
```

## 제외/포함 조건 문구

`인천(옹진군 제외)`처럼 괄호 안에 제외 또는 포함 조건이 있는 지역 문구는 나중에 자동 처리할 수 있도록 구조를 준비한다.
현재는 사람이 확인하되, 파싱 규칙으로 만들 수 있는 후보로 둔다.

예시:

```text
수집값=인천(옹진군 제외)
  -> 기본지역=인천
  -> 제외지역=인천,옹진군
  -> 현재는 사람 확인
```

예시:

```text
수집값=전국(제주 제외)
  -> 기본지역=전국
  -> 제외지역=제주
  -> 현재는 사람 확인
```

## 공사현장

공사현장은 적격심사평가세부기준에 지역평가점수가 있을 때 계산하기 위해 필요하다.
공사현장은 보통 공고문에 박혀 있으므로 현재는 사람이 직접 입력하는 값을 우선한다.
공사현장이 일반 지역명으로 들어오면 지역제한과 같은 방식으로 매핑한다.

예시:

```text
공고문=공사현장: 경기도 파주시 일원
사람입력 공사현장=경기,파주시
```

예시:

```text
공사현장=파주/포천
  -> 공사현장=경기,파주시/경기,포천시
```

공사현장에는 일반 지역명이 아니라 사업소 형태로 나오는 특수 발주처 값도 들어갈 수 있다.
이 값들은 별도 컬럼을 늘리지 않고 `공사현장`에 그대로 쓴다.
다만 일반 행정지역 마스터와는 분리된 사업소·특수지역 마스터에서 관리한다.

특수 유형:

```text
배전사업소
송전사업소
폐광지역진흥지구
```

표시 형식:

```text
사업소명은 그대로 쓴다.
여러 지역 또는 여러 사업소가 포함되면 `/`로 구분한다.
```

예시:

```text
공사현장=남서울본부/강남지사
  -> 공사현장=남서울본부/강남지사
```

예시:

```text
공사현장=폐광지역진흥지구
  -> 공사현장=폐광지역진흥지구
```

## 사업소·특수지역 마스터

사업소·특수지역은 일반 행정구역과 분리해 관리한다.
단, 화면 위치는 같은 `지역설정` 안에 둔다.

권장 구조:

```text
지역설정
  - 행정지역
  - 배전사업소
  - 송전사업소
  - 폐광지역진흥지구
```

### 배전사업소

파일:

```text
한전_배전사업소_2026-05-24.csv
```

컬럼:

| 컬럼 | 의미 | 예시 |
|---|---|---|
| id | 정렬/식별값 | `1` |
| 지역본부 | 한전 지역본부 | `서울` |
| 사업소명 | 공사현장에 표시할 사업소명 | `강남지사` |
| 인접 사업소 | 인접 사업소 목록 | `서초지사, 강동송파지사` |

예시:

```text
공사현장=강남지사
  -> 배전사업소 마스터에서 사업소명 매칭
  -> 공사현장=강남지사
```

인접 사업소 보정:

`지난번공고`, `이번공고`처럼 비교 메모가 섞인 값은 최종 마스터에 남기지 않는다.
사람이 확정한 최신 인접 사업소 목록만 `인접 사업소` 컬럼에 저장한다.

예시:

```text
사업소명=영암지사
인접 사업소=나주지사, 강진지사, 목포지사, 무안지사, 화순지사, 장흥지사, 해남지사
```

### 송전사업소

파일:

```text
송전사업소_2026-05-24.csv
```

컬럼은 배전사업소와 같다.
현재 파일은 헤더만 있고 데이터 행은 없다.
데이터가 들어오면 배전사업소와 같은 방식으로 관리한다.

예시:

```text
공사현장=○○송전지사
  -> 송전사업소 마스터에서 사업소명 매칭
  -> 공사현장=○○송전지사
```

### 폐광지역진흥지구

파일:

```text
폐광지역진흥지구_2026-05-24.csv
```

컬럼:

| 컬럼 | 의미 | 예시 |
|---|---|---|
| id | 정렬/식별값 | `1` |
| 광역 | 광역 표준값 | `강원` |
| 시·군 | 시군구 표준값 | `태백시` |
| 상세 구역 | 읍면동 또는 구역 설명 | `시 전역` |

예시:

```text
공사현장=폐광지역진흥지구
  -> 폐광지역진흥지구 마스터 참조
  -> 상세 대상 지역은 마스터에서 확인
```

## 검색용현장

검색용현장은 지역제한과 같은 방식으로 전처리한다.

예시:

```text
검색용현장=구리
  -> 경기,구리시
```

## 하위구/동읍 처리

하위구와 동읍은 특수한 경우에만 사용한다.
하위구가 매칭되면 상위 시군구를 찾아 최종 표시에는 상위 시군구까지만 쓴다.
하위구명 자체는 최종 `지역제한` 표시에 넣지 않는다.

예시:

```text
수집값=분당구
  -> 광역=경기
  -> 시군구=성남시
  -> 하위구=분당구
  -> 지역제한=경기,성남시
```

## 설정 화면 기준

지역명 표현은 계속 바뀔 수 있으므로 지역디비는 코드에 고정하지 않는다.

설정 화면에서 필요한 기능:

| 기능 | 이유 | 예시 |
|---|---|---|
| 광역 alias 수정 | 발주처별 표현 차이 대응 | `강원도`를 `강원특별자치도/강원/강원도`로 확장 |
| 시군구 alias 수정 | 짧은 이름 대응 | `구리` -> `구리시` |
| 하위구 alias 수정 | 특수 하위 행정구 대응 | `분당` -> `분당구` |
| 인접지역 수정 | 지역평가점수 계산 기준 변경 대응 | `구리시` 인접지역에 `남양주시` 추가 |
| 사업소 추가/수정 | 배전·송전 사업소 변경 대응 | `강남지사` 인접 사업소 수정 |
| 특수지역 수정 | 폐광지역진흥지구 변경 대응 | `보령시` 상세 구역 수정 |

## 아직 확인할 사항

### 지역디비 보조열 삭제

예시:

```text
지역디비 뒤쪽 `Unnamed: 15`, `Unnamed: 17`에 `수원`, `수원시` 같은 값이 반복되어 있다.
이 값은 보조열이므로 원본 파일에서 삭제한다.
```
