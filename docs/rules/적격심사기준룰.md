# 적격심사기준 룰

## 목적

공고분류 전처리의 핵심 목적은 해당 공고에 적용할 `적격심사기준`을 찾는 것이다.
적격심사기준을 찾지 못하면 심사 항목, 만점배수, 만점실적, 종목세부JSON을 확정할 수 없으므로 최종 분류도 할 수 없다.

예시:

```text
공고의 종목만점실적을 계산하려면
적격심사세부기준에서 만점배수를 찾고,
그 만점배수를 평가기준금액에 곱해야 한다.
평가기준금액은 기준에 따라 추정가격, 기초금액, 종목별 금액 등이 될 수 있다.
```

## 발주처와 적격심사기준

`발주처`는 입찰을 발주한 기관이다.
발주처마다 심사 기준이 다르며, 이 심사 기준을 찾기 위한 발주처 기준을 업무상 적격발주처 계열 컬럼으로 관리한다.

예시:

```text
한국전력공사가 발주한 공고라도 특정 공고의 심사기준은 조달청 기준일 수 있다.
```

## 세 가지 핵심 컬럼

### 적격발주처

`적격발주처`는 서버공고에 컬럼으로 존재할 수 있지만, 수집 단계에서 반드시 채워지는 값으로 보지 않는다.
값이 있으면 수집 출처와 발주처 홈페이지 연결 기준 후보로 사용하고, 빈값이면 전처리 또는 사람입력에서 보완한다.
내부적으로 해당 공고를 어느 홈페이지와 연결할지 판단하는 역할을 한다.

예시:

```text
`적격발주처=조달청`이면 나라장터 상세 화면으로 연결한다.
`적격발주처=한국전력공사`이면 한국전력공사 계열 공고 화면으로 연결한다.
```

주의:

```text
`적격발주처`는 서버공고에서 받을 수 있는 컬럼이지만 값이 비어 있을 수 있다.
실제 평가에 쓰는 심사기준은 `원발주처`에서 다시 확정한다.
```

### 원발주처

`원발주처`는 실제 평가에 사용할 적격심사기준 발주처다.
일반적으로 `적격발주처 = 원발주처`이지만 예외가 있다.

예시:

```text
홈페이지 기준은 `한국전력공사`이지만 실제 적격심사기준은 `조달청`일 수 있다.
이 경우 `적격발주처=한국전력공사`, `원발주처=조달청`처럼 분리한다.
```

발주처 내부에서도 특수한 공정은 별도 기준을 쓸 수 있다.
이 경우 괄호로 구분한다.

예시:

```text
일반 한국전력공사 기준 -> `한국전력공사`
배전특화공사 기준 -> `한국전력공사(배전)`
송전특화공사 기준 -> `한국전력공사(가공송전)`
```

### 적격평가기준_세부

`적격평가기준_세부`는 실제 A4 `적격심사세부기준` API를 호출하기 위한 세부 기준명이다.
`원발주처`를 찾은 뒤, 주공종 건설유형, 추정가격, 공고일/입력일을 기준으로 결정한다.
값은 원발주처와 적격심사기준을 `_`로 연결해 만든다.

예시:

```text
원발주처=행자부
적격심사기준=별표-4
  -> 적격평가기준_세부=행자부_별표-4
```

예시:

```text
원발주처=한국전력공사(배전)
적격심사기준=10억미만
  -> 적격평가기준_세부=한국전력공사(배전)_10억미만
```

`적격심사기준` 자체 표기는 원문을 보존한다.
발주처마다 `별표5`, `별표-5`, `50억미만`처럼 표기가 다를 수 있지만,
원문 기준명은 바꾸지 않고 A4 호출용 `적격평가기준_세부`에서만 조합해 사용한다.

예시:

```text
한국도로공사 적격심사기준=별표5
행자부 적격심사기준=별표-5
한국전력공사 적격심사기준=50억미만
  -> 원문 표기 그대로 보존
```

## 적격심사평가기준 매핑

적격심사평가기준 매핑은 최종 분류 자체가 아니라
A4 `적격심사세부기준`을 정확히 찾기 위한 전단계다.
따라서 이 단계는 특히 보수적으로 처리한다.
세부 테이블명과 화면 구조는 `docs/17_적격심사기준_분류구조_초안.md`에서 설계한 뒤 확정한다.
기존 인계 문서의 파일명과 함수명은 `docs/19_적격평가기준_세부_참고사항_정리.md`처럼 참고사항으로만 다룬다.

전체 구조는 두 번의 매칭으로 나눈다.

```text
1차 매칭: 적격심사평가기준 마스터에서 기본 적격심사기준을 찾는다.
2차 매칭: 세부조건 규칙표에서 공고확인 등 추가 조건을 확인하고 필요하면 덮어쓴다.
```

### 적격심사 제외 입찰

입찰방식이 적격심사 대상이 아니면 `적격평가기준_세부`를 빈값으로 둔다.
이는 오류가 아니라 정상 상태이며 흰색 셀로 표시한다.

예시:

```text
입찰방식=수의계약
  -> 적격평가기준_세부=빈값
  -> A4 적격심사세부기준 호출 안 함
```

예시:

```text
입찰방식=최저가
  -> 적격평가기준_세부=빈값
  -> 오류 표시 아님
```

오류는 적격심사기준을 찾아야 하는 공고인데 기준을 못 찾은 경우에만 표시한다.

예시:

```text
입찰방식=적격심사
원발주처=미등록발주처
  -> 기준을 찾아야 하는데 못 찾음
  -> 빨간 셀 또는 확인 알림
```

### 1차 매칭

1차 매칭은 적격심사평가기준 마스터에서 기본 적격심사기준을 찾는 단계다.
참고자료의 `master_evaluationCriteria.json`은 이 마스터의 한 구현 예시로만 본다.

매칭 기준:

```text
발주처 정규화 결과
주공종의 건설유형
추정가격
입력일 또는 공고일
```

금액 비교는 `추정가격`으로 한다.
`기초금액`이나 A4에서 나온 배수적용기준 금액이 아니라,
적격심사평가기준 행을 고르는 단계에서는 `추정가격`만 사용한다.

예시:

```text
원발주처=행자부
건설유형=일반건설
추정가격=450,000,000
기초금액=510,000,000
  -> 기준표 비교 금액은 450,000,000
```

금액 경계는 `미만`으로 본다.
기준 금액과 정확히 같으면 해당 구간에 넣지 않고 다음 구간을 확인한다.

예시:

```text
적격심사기준=2억미만
일반건설 (원)=200,000,000
추정가격=199,999,999
  -> 2억미만 적용

추정가격=200,000,000
  -> 2억미만 미적용, 다음 구간 확인
```

`시행일`은 공고일 또는 입력일과 비교해 어느 적격심사기준을 가져와야 하는지 판단하기 위해 쓴다.
시행일이 입력일 또는 공고일 이전인 행만 후보로 남기고, 그중 가장 최근 시행일 묶음을 사용한다.

예시:

```text
공고일=2026-02-01
시행일=2026-01-31
  -> 공고일 이후 이미 시행 중인 기준이므로 후보 가능

공고일=2026-01-20
시행일=2026-01-31
  -> 공고일 당시 아직 시행 전이므로 후보 제외
```

금액 구간은 기준 금액을 내림차순으로 훑으면서 후보를 갱신한다.
추정가격이 더 이상 기준 금액보다 작지 않은 순간 멈추고 직전 후보를 사용한다.

예시:

```text
발주처=한국전력공사
건설유형=일반건설
추정가격=250,000,000
입력일=2025-03-01

10억미만: 250,000,000 < 1,000,000,000 -> 후보
3억미만:  250,000,000 <   300,000,000 -> 후보 갱신
1억미만:  250,000,000 <   100,000,000 -> 아니오, 멈춤

1차 결과=한국전력공사_3억미만
```

복합 종목은 현재 단계에서는 주공종 기준으로만 적격심사평가기준을 고른다.
종목별 적격심사기준 개별 호출은 나중에 필요하면 별도 룰로 확장한다.

예시:

```text
단독평가종목=(건축,전기)
주공종=건축
건축=일반건설
전기=기타건설
  -> 현재는 주공종 건축의 일반건설 기준으로 적격심사평가기준을 고른다.
```

마스터에 없는 발주처가 나오면 배치 중 즉시 하나씩 끊지 않고, 수집 배치가 끝난 뒤 한 번에 모아 알린다.

예시:

```text
미등록 발주처 2개 발견:
- 경기도청
- 인천시청

적격심사평가기준 마스터에 추가 필요
```

### 2차 매칭

2차 매칭은 1차 결과를 키로 삼아 세부조건 규칙표를 다시 확인하는 단계다.
참고자료의 `orgCriteria`는 이 규칙표의 한 구현 예시로만 본다.
같은 1차 기준이라도 공고확인 같은 추가 조건에 따라 실제 원발주처와 적격평가기준_세부가 달라질 수 있기 때문이다.

예시:

```text
1차 결과=한국전력공사_3억미만
공고확인=배전공가
```

분류 규칙표:

| 매칭 키 | 공고확인 | 원발주처 | 적격평가기준_세부 |
|---|---|---|---|
| 한국전력공사_3억미만 | 배전공가 | 한국전력공사(배전공가) | 한국전력공사(배전공가)_3억미만 |

결과:

```text
원발주처=한국전력공사(배전공가)
적격평가기준_세부=한국전력공사(배전공가)_3억미만
```

조건 칸이 비어 있는 규칙은 조건 없이 통과하는 기본 규칙이다.

예시:

```text
매칭 키=조달청_별표-4
공고확인=빈값
  -> 조달청_별표-4에 해당하는 모든 공고에 적용 가능
```

2차 매칭은 멱등이어야 한다.
같은 공고에 여러 번 실행해도 같은 입력이면 같은 결과가 나와야 한다.

실행 시점:

```text
1. AI 수집 버튼 실행 후 문서곡괭이 매핑 직후 자동 실행
2. 사람이 공고확인 등 추가 정보를 입력하고 2차분류 완료 버튼을 누를 때 재실행
```

## 처리 순서

컬럼을 채우는 순서는 반드시 지켜야 한다.
종목 관련 기능은 거의 모든 공고 컬럼을 수집한 다음 진행한다.

```text
1. 서버공고에서 모든 컬럼을 받음
2. 곡괭이전나라시로 서버공고 raw 값만 보고 가능한 정리를 먼저 함
3. 적격발주처가 서버공고에서 비어 있으면 억지로 채우지 않음
4. 파싱용공고문과 문서곡괭이로 빈 컬럼 보완
5. 필요하면 사람 입력
6. 사람 입력 후 필요한 컬럼만 재전처리
7. 원발주처 변경사항이 있으면 다시 덮음
8. 적격심사 제외 입찰방식이면 적격평가기준_세부를 빈값으로 둠
9. 1차 매칭으로 기본 적격평가기준_세부 결정
10. 2차 매칭으로 공고확인 등 세부 조건을 반영해 덮어씀
11. A4 적격심사세부기준 API 호출
12. 기준 row에서 만점배수, 인정기간, 배수적용기준 등을 추출
13. 배수적용기준에 따라 평가기준금액 결정
14. 단독평가종목과 종목세부JSON 채움
15. 종목별 평가기준금액 x 만점배수로 종목만점실적 계산
16. 종목세부JSON과 적격기준을 참조하는 나머지 계산 컬럼 채움
17. 최종 출력
```

## 왜 2차 전처리가 필요한가

수집 단계에서 모든 컬럼이 완벽하게 채워지면 가장 좋다.
하지만 실제로는 `원발주처`, `건설유형`, `전문건설_주력분야`, `단독평가종목` 같은 값이 한 번에 완벽하게 채워지지 않을 수 있다.
그래서 사람 입력 후 필요한 컬럼만 다시 전처리한다.

예시:

```text
1차 전처리에서 `원발주처=한국전력공사`로 잡힘
사람이 공고 내용을 보고 `원발주처=한국전력공사(배전)`으로 수정
  -> `적격평가기준_세부`, A4 기준 row, 종목만점실적, 종목세부JSON만 다시 계산
```

## 종목만점실적 계산 방향

종목만점실적은 적격심사기준의 만점배수를 찾아 평가기준금액에 곱해서 만든다.

기본 구조:

```text
종목만점실적 = 평가기준금액 x 만점배수
```

평가기준금액은 적격심사세부기준의 배수적용기준에 따라 달라진다.
추정가격일 수도 있고, 기초금액일 수도 있고, 종목별 금액일 수도 있다.

예시:

```text
배수적용기준=`추정가격`
추정가격=1,000,000,000
평가기준금액=1,000,000,000
만점배수=1.0
  -> 종목만점실적=1,000,000,000
```

예시:

```text
배수적용기준=`기초금액`
기초금액=800,000,000
평가기준금액=800,000,000
만점배수=0.8
  -> 종목만점실적=640,000,000
```

예시:

```text
단독평가종목=(토목,전기)
토목 평가기준금액=800,000,000
전기 평가기준금액=200,000,000
만점배수=1.0
  -> 토목 종목만점실적=800,000,000
  -> 전기 종목만점실적=200,000,000
```

주의:

```text
현재 적격심사평가기준은 주공종 기준으로만 고른다.
복합 공고에서 종목별 평가기준금액이 달라 개별 A4 호출이 필요해 보이는 경우는
향후 확장 후보로 메모만 남긴다.
예: 토목 8억, 전기 2억이면 토목 기준과 전기 기준의 A4 호출 결과가 달라질 수 있다.
```

## A4 API와의 연결

A4 `적격심사세부기준` API는 `적격평가기준_세부`와 `건설` 값을 기준으로 호출한다.

예시:

```text
적격평가기준_세부=`행자부_별표-4`
건설=`일반건설`
  -> A4에서 해당 기준 row 목록을 가져온다.
```

가져온 기준 row에서 아래 값을 추출해 종목세부JSON과 계산 컬럼을 채운다.
원본 A4 row 이름이 `업종실적`이면 업무상 `종목실적`으로 읽는다.

```text
시행일
종목실적 인정기간
종목실적 배수적용기준
종목실적 만점배수
동일실적 인정기간
동일실적 배수적용기준
동일실적 만점배수
```

### 상세모달 `적격심사기준` 탭 표시

공고관리 상세모달의 `적격심사기준` 탭은 A4 row를 그대로 엑셀 표처럼 보여주는 화면이 아니다.
사용자가 심사 항목 구조를 확인할 수 있게 `메모`와 `이름` 관계를 이용해 아코디언 트리로 표시한다.
`적격발주처`, `원발주처`, `적격평가기준_세부`, 1차/2차 매칭상태 같은 입력·진단 필드는 `기본` 탭에서 확인하므로 이 탭에 중복 표시하지 않는다.

호출 흐름:

```text
상세모달에서 `적격심사기준` 탭 클릭
  -> row.적격평가기준_세부 확인
  -> row.일반_기타_전문 확인
  -> /api/qualification?적격기준세부=...&건설=... 호출
  -> 시행일을 공고일/입력일 기준으로 필터
  -> 메모가 부모명, 이름이 항목명인 트리로 복원
  -> 좁은 우측 패널에서 밀리지 않도록 최소 들여쓰기 아코디언으로 표시
```

예시:

```text
적격평가기준_세부=행자부_별표-4
일반_기타_전문=일반건설
입력일=2026-05-22

호출:
/api/qualification?적격기준세부=행자부_별표-4&건설=일반건설

표시:
적격통과점수
입찰가격점수
수행능력평가
  └ 업종실적
  └ 동일실적
```

주의:

```text
적격평가기준_세부가 빈값이면 A4를 호출하지 않는다.
건설유형이 빈값이면 기본값으로 일반건설을 쓸 수 있지만, 실제 분류에서는 건설유형을 먼저 채우는 것이 원칙이다.
A4 응답의 시행일이 여러 개면 공고일/입력일 이전의 가장 최근 시행일 묶음을 사용한다.
```

## 보수적 원칙

적격심사기준을 잘못 찾으면 이후 계산이 모두 틀어진다.
따라서 확실하지 않은 경우에는 임의로 채우지 않고 사람 입력 또는 확인 대상으로 남긴다.

예시:

```text
공고문에 배전특화공사인지 애매하면
`원발주처=한국전력공사(배전)`으로 확정하지 않고 확인 대상으로 둔다.
```
