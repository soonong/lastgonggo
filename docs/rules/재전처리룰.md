# 재전처리 룰

## 기본 원칙

사람이 입력한 값 때문에 영향을 받는 컬럼만 다시 처리한다.

## 초기 의존관계

| 사람이 수정한 컬럼 | 재처리 대상 컬럼 | 이유 | 예시 |
|---|---|---|---|
| 공고명 | 공고분류, 종목그룹, 키워드 | 공고명이 분류 기준이 되기 때문 | `급식실 개선공사` -> 시설 공사 후보 |
| 발주처명 | 발주처표준명, 기관구분, 지역 | 발주처 기준값이 바뀌기 때문 | `서울교육청` -> `서울특별시교육청` |
| 종목 | 종목그룹, 공고분류, 단독평가종목, 종목세부JSON | 종목이 분류와 심사 기준이 되기 때문 | `기타` -> `전기` |
| 전문건설_주력분야 | 단독평가종목, 종목세부JSON, 종목별 평가기준금액, 종목만점실적 관련 컬럼 | 주력분야가 바뀌면 실제 평가종목이 바뀌기 때문 | `포장` -> `토공` |
| 단독평가종목 | 종목세부JSON, 종목별 평가기준금액, 종목만점실적 관련 컬럼 | 단독평가종목이 실제 평가 단위이기 때문 | `철콘_건축` -> `철콘_토목` |
| 기초금액 | 금액구간, 예산분류 | 금액 기반 계산이 필요하기 때문 | `120000000` -> `1억 이상` |
| 추정가격 | 1차 적격심사평가기준, 적격평가기준_세부, A4 기준 row | 적격심사평가기준 매칭의 금액 비교 기준이기 때문 | `450000000` -> `행자부_별표-5` |
| 공고확인 | 2차 적격심사기준 매칭, 원발주처, 적격평가기준_세부 | 세부 조건에 따라 실제 기준이 바뀔 수 있기 때문 | `배전공가` -> `한국전력공사(배전공가)_3억미만` |
| 평가기준금액 | 종목세부JSON 금액, 종목만점실적 | 만점실적은 평가기준금액 x 만점배수로 계산하기 때문 | `800000000` -> `토목 종목만점실적=800000000` |

## 보호 규칙

사람이 수정한 컬럼 자체는 재전처리 결과가 덮어쓰지 않는다.

예시:

```text
사람 입력 `종목=전기`는 보호한다.
재전처리는 `종목그룹`, `공고분류`, `단독평가종목`, `종목세부JSON`처럼
종목에서 파생되는 컬럼만 갱신한다.
```

예시:

```text
사람 입력 `전문건설_주력분야=토공`은 보호한다.
재전처리는 `단독평가종목=지반조성포장_토공`,
`종목세부JSON`, `종목만점실적`처럼 파생 컬럼만 다시 갱신한다.
```

예시:

```text
상호진출 자동값이 `철콘_건축`으로 들어갔지만 사람이 `철콘_토목`으로 직접 수정하면
`단독평가종목=철콘_토목`은 보호한다.
재전처리는 `종목세부JSON`, `종목별 평가기준금액`, `종목만점실적`만 다시 갱신한다.
```
