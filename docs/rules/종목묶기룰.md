# 종목묶기 룰

## 목적

종목묶기는 공고에서 수집한 종목을 화면 표시용 `종목`과 실제 심사용 `단독평가종목`으로 나누어 만드는 규칙이다.

```text
원본 종목 표현
  -> 종목 매핑
  -> 표시용 종목
  -> 심사용 단독평가종목
  -> 종목세부JSON
```

상세 케이스는 `종목묶기_케이스매트릭스.md`에서 관리한다.
이 문서는 원칙을 설명하고, 케이스 매트릭스는 실제 입력/출력 예시와 확정 상태를 관리한다.

예시:

```text
`건축/토목 -> (건축,토목)/토건` 같은 개별 케이스는
docs/rules/종목묶기_케이스매트릭스.md의 케이스 ID로 추적한다.
```

## 용어 기준

앞으로 업무 문서에서는 `업종`이라는 표현을 `종목`으로 바꿔 쓴다.
단, 첨부 CSV의 원본 컬럼명이 `업종`, `주력업종`, `상위업종`, `대체업종`인 경우에는 아래처럼 읽는다.

| 파일 컬럼명 | 업무 용어 |
|---|---|
| 업종 | 종목 |
| 주력업종 | 주력종목 |
| 상위업종 | 상위종목 |
| 대체업종 | 대체종목 |

예시:

```text
종목_매핑 CSV의 `업종=금속창호지붕`, `주력업종=금속구조물`은
업무상 `종목=금속창호지붕`, `주력종목=금속구조물`로 읽는다.
```

## 종목과 단독평가종목의 차이

`종목`은 홈페이지에서 사용자가 보는 표시용 컬럼이다.
심사용이 아니라 복잡한 평가종목을 단순하게 보여주는 용도다.

`단독평가종목`은 실제 적격심사기준에서 평가 기준이 되는 종목이다.
종목세부JSON은 단독평가종목을 종목별 객체로 풀어낸 결과다.

| 컬럼 | 용도 | 들어가는 값 |
|---|---|---|
| 종목 | 사용자 표시용 | 일반/기타건설 종목, 전문건설 대업종 |
| 단독평가종목 | 실제 심사용 | 일반/기타건설 종목, 전문건설 대업종_주력종목, 상호진출 평가종목. 단 `토건`은 제외 |
| 종목세부JSON | 계산/상세용 | 단독평가종목별 세부 객체 |

예시:

```text
공고문: 금속창호지붕(주력분야: 금속구조물)
종목: 금속창호지붕
단독평가종목: 금속창호지붕_금속구조물
종목세부JSON: 금속창호지붕_금속구조물 객체
```

## 전문건설 주력종목 규칙

전문건설은 대업종과 주력종목을 가진다.
단독평가종목은 기본적으로 `대업종_주력종목` 형태로 만든다.
수집/파싱 단계에서 먼저 `종목=대업종`, `전문건설_주력분야=주력종목`으로 변환한 뒤 종목묶기를 한다.

예시:

```text
종목=금속창호지붕
주력종목=금속구조물
  -> 단독평가종목=금속창호지붕_금속구조물
```

주력종목이 없는 전문건설도 있다.
종목 매핑표에서 `종목`과 `주력종목`이 같으면 주력종목은 없는 것으로 보고 삭제한다.
이 경우 단독평가종목은 대업종인 종목 그대로 사용한다.

예시:

```text
종목=철콘
주력종목=철콘
  -> 주력종목 삭제
  -> 단독평가종목=철콘
```

예시:

```text
공고문에 `포장공사업`이 있으면
  -> 종목=지반조성포장
  -> 전문건설_주력분야=포장
  -> 단독평가종목=지반조성포장_포장
```

예시:

```text
공고문에 `포장공사업/토공사업`, 종목모두보유=0이면
  -> 종목=지반조성포장
  -> 단독평가종목=지반조성포장_포장/지반조성포장_토공

공고문에 `포장공사업/토공사업`, 종목모두보유=1이면
  -> 종목=지반조성포장
  -> 단독평가종목=(지반조성포장_포장,지반조성포장_토공)
```

## 기호 의미

최종 표기에서 기호는 아래 의미를 가진다.

| 기호 | 의미 | 예시 |
|---|---|---|
| `,` | 같이 묶임. 한 공고에서 모두 필요 | `(건축,토목)`은 건축과 토목 둘 다 필요 |
| `/` | 다른 옵션. 어느 한 줄로 응찰 가능 | `건축/토목`은 건축 또는 토목 중 하나 |

예시:

```text
(건축,토목)
  -> 건축 면허와 토목 면허 둘 다 필요

건축/토목
  -> 건축 면허 또는 토목 면허 중 하나로 가능

(건축,토목)/토건
  -> 건축+토목 둘 다 있거나, 토건 하나가 있거나
```

## 상위종목

현재 확정된 상위종목은 2개다.

| 하위 종목 | 상위종목 |
|---|---|
| 건축, 토목 | 토건 |
| 기계소방, 전기소방 | 전문소방 |

상위종목은 사용자가 보는 종목 표기와 평가 가능 후보에 영향을 준다.
단, `토건`은 어디서든 단독평가종목에 넣지 않는다.

예시:

```text
건축 -> 건축/토건
단독평가종목 -> 건축
토목 -> 토목/토건
단독평가종목 -> 토목
건축/토목 -> (건축,토목)/토건
단독평가종목 -> (건축,토목)
건축/토목/토건 -> (건축,토목)/토건
단독평가종목 -> (건축,토목)
```

예시:

```text
기계소방 -> 기계소방/전문소방
전기소방 -> 전기소방/전문소방
기계소방/전기소방 -> (기계소방,전기소방)/전문소방
기계소방/전기소방/전문소방 -> (기계소방,전기소방)/전문소방
```

주의:

```text
단일 종목은 일반적으로 입력을 유지하지만,
상위종목이 있는 `건축`, `토목`, `기계소방`, `전기소방`은 상위종목 옵션을 추가한다.
```

## 대체종목

대체종목은 참여 가능 면허 매칭에서 자동 인정한다.
표시용 `종목`에는 원 종목과 대체종목을 슬래시로 함께 표시한다.
심사용 `단독평가종목`은 원 종목 기준을 유지한다.

예시:

```text
산림사업_토목 = 산림조합_지역
  -> 회사 면허가 산림조합_지역이면 산림사업_토목 공고 매칭 가능으로 본다
```

예시:

```text
기계가스설비 -> 가스난방
  -> 회사 면허가 가스난방이면 기계가스설비 공고 매칭 가능으로 본다
  -> 표시용 종목은 기계가스설비/가스난방
```

## 참조 컬럼

종목묶기는 아래 컬럼을 참조한다.

```text
종목모두보유
상호진출여부
일반_기타_전문
전문건설_주력분야
참여형태
공동도급형태
```

예시:

```text
상호진출여부=1이면 일반건설과 전문건설이 서로 참여하는 상호진출 규칙을 적용한다.
전문건설_주력분야=금속구조물이면 전문건설 단독평가종목을 `금속창호지붕_금속구조물`로 만든다.
```

## 종목모두보유와 공동도급 포지션

`종목모두보유`, `공동이행`, `분담이행`, `주계약자관리`는 종목묶기에서 같은 포지션으로 본다.
이 포지션이면 하위 종목을 같이 필요한 묶음으로 보고, 상위종목 옵션을 별도로 만든다.

예시:

```text
입력=건축/토목
종목모두보유=1
  -> 종목=(건축,토목)/토건
```

예시:

```text
입력=건축/토목
종목모두보유=0
  -> 종목=건축/토목/토건
```

예시:

```text
입력=건축/전기
공동도급형태=분담이행 또는 주계약자관리
  -> 종목=(건축,전기)/(토건,전기)
```

예시:

```text
입력=건축/전기
참여형태=협정, 공동도급형태=공동이행
  -> 종목=(건축,전기)/(토건,전기)
```

## 기본 생성 단계

종목묶기는 아래 순서로 처리한다.

```text
1. 원본 종목 문자열을 분리한다.
2. 종목 매핑표로 각 원본 표현을 표준 종목 객체로 바꾼다.
3. 수집/파싱값을 우선해 종목, 전문건설_주력분야, 일반_기타_전문을 정규화한다.
4. 전문건설에서 종목과 주력종목이 같으면 주력종목을 삭제한다.
5. 표시용 종목 단위를 만든다.
6. 심사용 단독평가종목 단위를 만든다.
7. 상위종목 옵션을 추가한다.
8. 상호진출여부에 따라 일반 규칙 또는 상호진출 규칙을 적용한다.
9. 최종 종목 표기와 단독평가종목 표기를 만든다.
10. 단독평가종목을 기준으로 종목세부JSON을 만든다.
```

주의:

```text
종목조건없음은 종목명으로 반환하지 않는다.
최종 종목과 단독평가종목은 모두 빈값을 반환한다.
입력순서는 종목 표시와 단독평가종목 표시에서 보존한다.
```

## 상호진출=0

상호진출이 아니면 입력 종목들은 같은 공고에서 필요한 종목 묶음으로 보고 처리한다.
주공종은 입력의 맨 앞 종목이다.

### 일반 주공종

| 조합 | 입력 예시 | 종목 | 단독평가종목 |
|---|---|---|---|
| 일반+일반 | 건축/토목 | `(건축,토목)/토건` | `(건축,토목)` |
| 일반+기타 | 건축/전기 | `(건축,전기)/(토건,전기)` | `(건축,전기)` |
| 일반+기타×2 | 토목/전기/통신 | `(토목,전기,통신)/(토건,전기,통신)` | `(토목,전기,통신)` |
| 일반+전문 | 건축/실내건축 | `(건축,실내건축)/(토건,실내건축)` | `(건축,실내건축)` |
| 일반+전문, 주력 일부 | 토목/토공/포장 | `(토목,지반조성포장)/(토건,지반조성포장)` | `(토목,지반조성포장_토공,지반조성포장_포장)` |
| 일반+전문, 주력 각 1개 | 토목/토공/기계설비 | `(토목,지반조성포장,기계가스설비)/(토건,지반조성포장,기계가스설비)` | `(토목,지반조성포장_토공,기계가스설비_기계설비)` |
| 일반+기타+전문 | 건축/전기/조경식재 | `(건축,전기,조경식재시설물)/(토건,전기,조경식재시설물)` | `(건축,전기,조경식재시설물_조경식재)` |

### 기타 주공종

| 조합 | 입력 예시 | 종목 | 단독평가종목 |
|---|---|---|---|
| 기타+기타, 상위 적용 | 전기/전기소방 | `(전기,전기소방)/(전기,전문소방)` | `(전기,전기소방)/(전기,전문소방)` |
| 기타+일반 | 전기/건축 | `(전기,건축)/(전기,토건)` | `(전기,건축)` |
| 기타+전문, 상위 적용 | 기계소방/기계설비 | `(기계소방,기계가스설비)/(전문소방,기계가스설비)` | `(기계소방,기계가스설비_기계설비)/(전문소방,기계가스설비_기계설비)` |
| 기타+기타+전문 | 전기/전문소방/금속구조물/지붕판금 | `(전기,전문소방,금속창호지붕)` | `(전기,전문소방,금속창호지붕)` |

### 전문 주공종

| 조합 | 입력 예시 | 종목 | 단독평가종목 |
|---|---|---|---|
| 전문+전문 | 금속구조물/철콘 | `(금속창호지붕,철콘)` | `(금속창호지붕_금속구조물,철콘)` |
| 전문+기타 | 조경식재/전기 | `(조경식재시설물,전기)` | `(조경식재시설물_조경식재,전기)` |
| 전문+일반 | 실내건축/건축 | `(실내건축,건축)/(실내건축,토건)` | `(실내건축,건축)` |
| 전문+일반+기타 | 포장/전기/건축 | `(지반조성포장,전기,건축)/(지반조성포장,전기,토건)` | `(지반조성포장_포장,전기,건축)` |

## 상호진출=1

상호진출은 일반건설과 전문건설이 서로의 영역에 참여하는 경우다.
기타건설은 상호진출 대상이 아니며, 필요한 경우 일반/전문 묶음에 같이 따라간다.

### 종합공사: 일반 주공종 + 전문 참여

원칙:

```text
전문공사의 단독평가종목을 단독으로 넣지 않는다.
전문 파트는 반드시 일반 주공종을 접미사로 붙인다.
```

예시:

```text
전문 단독평가종목 `실내건축`
일반 주공종 `건축`
  -> `실내건축_건축`
```

| 조합 | 입력 예시 | 종목 | 단독평가종목 |
|---|---|---|---|
| 일반+전문 | 건축/실내건축 | `건축 / 토건 / 실내건축` | `건축 / 실내건축_건축` |
| 일반+전문×2 | 건축/실내건축/도장 | `건축 / 토건 / (실내건축,도장습식방수석공)` | `건축 / (실내건축_건축,도장습식방수석공_도장_건축)` |
| 일반+전문+기타 | 토목/전기/토공 | `(토목,전기) / (토건,전기) / (지반조성포장,전기)` | `(토목,전기) / (지반조성포장_토공_토목,전기)` |

### 전문공사: 전문 주공종 + 일반 참여

원칙:

```text
일반건설 참여 종목은 상호진출 매핑표에 근거해 전문 주공종 단독평가종목을 접미사로 붙인다.
토건은 종목 표시에는 나오지만 단독평가종목에서는 평가하지 않으므로 넣지 않는다.
```

예시:

```text
전문 주공종 `지반조성포장_토공`
일반 참여 종목 `토목`
  -> `토목_지반조성포장_토공`
```

주의:

```text
상호진출 접미사는 단순히 첫 종목을 붙이는 규칙이 아니다.
종목매핑 마스터의 `상호진출_상대종목` 컬럼을 근거로 붙인다.
예: 건축/토목/실내건축에서 실내건축은 매핑표 기준으로 `실내건축_건축`이 된다.
`실내건축_(건축,토목)`처럼 묶음 전체를 접미사로 붙이는 케이스는 없다.
철콘처럼 건축과 토목 양쪽에 모두 매핑되는 경우에는 입력순서 기준 첫 매칭을 자동값으로 두고, 사람이 수정할 수 있게 둔다.
```

## 상호진출 상대종목

상호진출 상대종목은 별도 파일로 분리하기보다 종목매핑 마스터에서 함께 관리한다.
컬럼명은 `상호진출_상대종목`으로 둔다.

예시:

```text
일반종목=건축
상호진출_상대종목=실내건축, 금속창호지붕, 도장습식방수석공, 비계구조, 철콘, 지반조성포장
```

예시:

```text
일반종목=토목
상호진출_상대종목=지반조성포장, 상하수도, 철도궤도, 수중준설, 승강기삭도, 철콘, 비계구조
```

확정된 1차 매핑:

| 일반종목 | 상호진출_상대종목 |
|---|---|
| 건축 | 실내건축, 금속창호지붕, 도장습식방수석공, 비계구조, 철콘, 지반조성포장 |
| 토목 | 지반조성포장, 상하수도, 철도궤도, 수중준설, 승강기삭도, 철콘, 비계구조 |
| 산업환경설비 | 기계가스설비, 상하수도, 수중준설, 가스난방 |
| 조경 | 조경식재시설물, 지반조성포장 |

| 조합 | 입력 예시 | 종목 | 단독평가종목 |
|---|---|---|---|
| 전문+일반 | 토공/토목 | `지반조성포장 / 토목 / 토건` | `지반조성포장_토공 / 토목_지반조성포장_토공` |
| 전문×2+일반×2 | 금속구조물/습식방수/건축/토목 | `(금속창호지붕,도장습식방수석공) / (건축,토목) / 토건` | `(금속창호지붕_금속구조물,도장습식방수석공_도장) / (건축_금속창호지붕_금속구조물,토목_금속창호지붕_금속구조물)` |
| 전문+일반+기타 | 토공/전기/토목 | `(지반조성포장,전기) / (토목,전기) / (토건,전기)` | `(지반조성포장_토공,전기) / (토목_지반조성포장_토공,전기)` |

## 로직 설계 초안

구현 시 함수는 크게 나눈다.

| 단계 | 함수 역할 | 예시 |
|---|---|---|
| 매핑 | 원본 표현을 종목 객체로 변환 | `토공 -> 전문건설, 종목=지반조성포장, 주력종목=토공` |
| 정규화 | 주력종목 삭제, 중복 제거, 상위종목 정보 정리 | `종목=철콘, 주력종목=철콘 -> 주력종목 없음` |
| 표시종목 생성 | 사용자 표시용 종목 단위 생성 | `토공 -> 지반조성포장` |
| 평가종목 생성 | 단독평가종목 단위 생성 | `토공 -> 지반조성포장_토공` |
| 상위종목 확장 | 토건/전문소방 옵션 생성 | `건축,토목 -> (건축,토목)/토건` |
| 상호진출 처리 | 일반/전문 접미사 규칙 적용 | `실내건축 + 건축 -> 실내건축_건축` |
| 직렬화 | 괄호, 쉼표, 슬래시 표기 생성 | `(건축,전기)/(토건,전기)` |
| JSON 전개 | 단독평가종목을 종목세부JSON으로 변환 | 종목별 객체 생성 |

의사코드:

```text
buildJongmok(rawTokens, context):
  mapped = mapRawTokens(rawTokens, jongmokMapping, sortByLengthDesc=true)
  normalized = normalizeMappedItems(mapped)
  if context.상호진출여부 == 1:
    result = buildCrossMarketGroups(normalized, context)
  else:
    result = buildNormalGroups(normalized, context)
  result.종목세부JSON = expandToDetailJson(result.단독평가종목, context.적격심사기준)
  return result
```

## 종목세부JSON 전개

종목세부JSON은 `단독평가종목`의 구분자 의미를 보존하지 않는다.
슬래시, 괄호, 쉼표를 해석한 뒤 앞에서부터 순서대로 모든 종목을 넣는다.
단독평가종목에 `a/a`, `(a,b)/(a,b)`처럼 완전히 같은 옵션이 반복되면 값이 꼬인 것으로 보고 알림을 띄운다.
단, `(a,b)/(a,c)`처럼 일부 종목만 겹치고 옵션 자체가 다르면 정상으로 보고 순서대로 넣는다.

예시:

```text
단독평가종목=a/(b,c)/d
  -> 종목세부JSON 순서=a, b, c, d
```

예시:

```text
단독평가종목=(a,b)/(a,c)
  -> 종목세부JSON 순서=a, b, a, c
  -> 서로 다른 옵션이므로 알림 대상이 아님

단독평가종목=(a,b)/(a,b)
  -> 알림: 같은 옵션 반복 확인 필요
```

예시:

```text
단독평가종목=지반조성포장_포장/지반조성포장_토공
  -> 종목세부JSON 순서=지반조성포장_포장, 지반조성포장_토공

단독평가종목=(지반조성포장_포장,지반조성포장_토공)
  -> 종목세부JSON 순서=지반조성포장_포장, 지반조성포장_토공
```

## 종목세부JSON 금액

종목세부JSON의 금액 기준은 `평가기준금액`이다.
평가기준금액은 적격심사세부기준의 배수적용기준에 따라 추정가격, 기초금액, 종목별 금액 등으로 달라질 수 있다.

기본 원칙:

```text
비율은 기본 계산하지 않는다.
정확성을 위해 금액 기준으로 계산한다.
공고문에 금액이 없고 비율만 있을 때는 평가기준금액에서 역산할 수 있다.
```

예시:

```text
단독평가종목=지반조성포장_포장/지반조성포장_토공
평가기준금액=1억
  -> 지반조성포장_포장 평가기준금액=1억
  -> 지반조성포장_토공 평가기준금액=1억
```

예시:

```text
단독평가종목=(토목,전기)
토목 금액=8억
전기 금액=2억
  -> 토목 평가기준금액=8억
  -> 전기 평가기준금액=2억
  -> 비율은 계산하지 않음
```

예시:

```text
단독평가종목=(토목,전기)
평가기준금액=10억
종목별 금액 없음
  -> 첫 번째 종목 토목에 10억을 넣음
  -> 전기 평가기준금액은 빈값으로 둠
  -> 이후 사람이 직접 채울 수 있음
```

예시:

```text
단독평가종목=(토목,전기)
평가기준금액=10억
공고문 비율=토목 80%, 전기 20%
  -> 토목 평가기준금액=8억
  -> 전기 평가기준금액=2억
```

주의:

```text
비율 역산값에 별도 산출근거 메모를 자동으로 만들지 않는다.
확실하게 계산할 수 있으면 금액만 채우고, 확실하지 않으면 빈값으로 둔다.
빈값은 사람이 확인해서 채운다.
```

주의:

```text
대체종목은 종목과 단독평가종목 생성 단계에서 이미 처리된다.
종목세부JSON은 단독평가종목만 보고 생성한다.
토건은 단독평가종목에 없으므로 종목세부JSON에도 없다.
전문소방은 단독평가종목에 있으면 종목세부JSON에도 들어간다.
```

## 보수적 원칙

종목묶기는 적격심사기준, 종목만점실적, 참여 가능성 판단에 직접 영향을 준다.
따라서 애매한 값은 임의로 만들지 않고 확인 대상으로 둔다.

예시:

```text
원본문자 `포장`만 보고 확정하지 않는다.
수집/파싱 후 `종목=지반조성포장`, `전문건설_주력분야=포장`으로 정규화되면
단독평가종목은 `지반조성포장_포장`으로 확정한다.
```

## 설정 구조

상호진출 상대종목은 종목매핑 파일의 `상호진출_상대종목` 컬럼으로 관리한다.
별도 하위 테이블로 분리하지 않는다.

예시:

```text
종목=건축
상호진출_상대종목=실내건축,금속창호지붕,도장습식방수석공,비계구조,철콘,지반조성포장
```
