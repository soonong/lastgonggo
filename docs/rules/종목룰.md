# 종목 룰

## 목적

`종목`, `단독평가종목`, `종목세부JSON`은 한 묶음으로 관리한다.

```text
종목
  -> 단독평가종목
  -> 종목세부JSON
```

이 세 컬럼은 입찰에 필요한 면허를 표준화하고, 평가 가능한 단위로 분해한 뒤, 종목별 세부 정보를 화면과 계산에 넘기기 위한 핵심 흐름이다.

## 종목의 의미

종목은 입찰공고에서 정한 공사 참여 자격이다.
쉽게 말하면 "이 공사를 하려면 어떤 면허가 있어야 하는가"를 나타낸다.

예시:

```text
초등학교 강당 짓기 -> 종목=건축
도로 포장하기 -> 종목=포장 또는 토목
병원 전기설비 공사 -> 종목=전기
```

발주처가 종목을 쓰는 방식은 제각각이므로, 문서곡괭이와 룰은 이를 표준 종목명으로 맞춘다.

예시:

```text
`0039(전기)`, `전기공사업 등록업체`, `종목: 전기`,
`법령에 의한 전기공사업 등록업체`는 모두 표준 종목 `전기`로 맞춘다.
```

## 건설유형

시설공사의 종목은 크게 세 그룹으로 나눈다.

| 건설유형 | 의미 | 예시 |
|---|---|---|
| 일반건설 | 종합공사, 큰 공사 | `건축`, `토목`, `토건`, `조경`, `산업환경설비` |
| 기타건설 | 일반/전문에 속하지 않는 특수 분야 | `전기`, `통신`, `전문소방`, `기계소방`, `전기소방` |
| 전문건설 | 부분 공사 전문 | `실내건축`, `금속구조물`, `도장`, `포장`, `비계`, `철콘` |

예시:

```text
아파트 신축은 일반건설 `건축`으로 본다.
CCTV 설치는 기타건설 `통신`으로 본다.
아파트 인테리어는 전문건설 `실내건축`으로 본다.
```

## 표준 종목 카탈로그

공고문에서 파싱한 종목은 표준 종목 카탈로그에 있는 약칭으로 통일한다.

예시:

```text
공고문 `전기공사업 등록업체`
  -> 표준 종목 `전기`
  -> 회사 보유 면허 `전기`와 매칭 가능
```

현재 기준 설명에서는 `data/jongmok.json`의 약 35종 약칭을 표준값으로 본다.
실제 웹 구현에서는 이 기준 파일을 설정 화면에서 추가, 변경, 수정할 수 있어야 한다.

## 상위종목

일부 종목 조합은 상위종목으로 인정된다.
상위종목은 자동으로 추가할 수 있다.

| 조합 | 상위종목 | 예시 |
|---|---|---|
| 건축 + 토목 | 토건 | `건축/토목`이면 `건축`, `토목`, `토건` 모두 인정 |
| 기계소방 + 전기소방 | 전문소방 | `기계소방/전기소방`이면 `전문소방` 인정 |

예시:

```text
입찰공고 종목이 `건축/토목`이면 시스템은 `건축`, `토목`, `토건`을 후보 종목으로 본다.
```

## 전문건설 대업종과 주력종목

전문건설은 비슷한 종목끼리 묶은 `대업종`과 그 안의 `주력종목`을 가진다.
기존 파일이나 화면에 `주력업종`이라고 적힌 값은 업무상 `주력종목`으로 읽는다.

예시:

```text
대업종 `금속창호지붕`
  -> 주력종목 `금속구조물`
  -> 주력종목 `지붕판금`
```

표기 예시:

```text
회사 또는 공고가 `금속구조물`만 요구하면 `금속창호지붕_금속구조물`로 표기한다.
`금속구조물`과 `지붕판금`을 모두 요구하면 대업종 `금속창호지붕`으로 합쳐 표기할 수 있다.
```

## 상호진출

상호진출은 일반건설과 전문건설이 서로 다른 영역의 공사에 참여할 수 있는 경우를 말한다.

기본 원칙:

```text
일반건설은 종합공사
전문건설은 부분공사
기타건설은 상호진출 대상 아님
```

예시:

```text
건축 공사인데 실내건축 전문건설 회사도 응찰 가능하면
종목에 `건축/실내건축`이 같이 나타날 수 있다.
```

주의:

```text
전기, 통신, 소방 같은 기타건설은 상호진출로 보지 않는다.
```

## 단독평가종목

`단독평가종목`은 원본 종목을 평가 가능하게 분해했다가 재조립한 값이다.
규칙은 복잡하므로 종목 묶기와 분해 기준은 `종목묶기룰.md`에서 별도로 관리한다.

예시:

```text
원본 종목 `토목, 산업환경설비`
  -> 평가 가능하도록 비율과 기준금액을 나눔
  -> 단독평가종목 `(토목,산업환경설비)` 생성
```

## 종목세부JSON

`종목세부JSON`은 `단독평가종목`을 종목별 객체로 풀어낸 값이다.
단독평가종목과 종목세부JSON은 한몸으로 본다.

예시:

```json
[
  {
    "종목": "토목",
    "종목평가기준금액": 5161703596,
    "종목비율": 0.845501605349538,
    "종목실적평가기간": 5
  },
  {
    "종목": "산업환경설비",
    "종목평가기준금액": 943197404,
    "종목비율": 0.15449839465046197,
    "종목실적평가기간": 5
  }
]
```

## 동일실적_평가여부

`동일실적_평가여부`는 동일실적 컬럼에 평가비율이 있으면 `1`, 없으면 `0`이다.

예시:

```text
종목세부JSON 또는 동일실적 컬럼 묶음에 동일실적 평가비율이 있으면 `동일실적_평가여부=1`이다.
동일실적 평가비율이 없으면 `동일실적_평가여부=0`이다.
```

## 전문건설_주력분야

`전문건설_주력분야`는 서버공고에서 받은 값 또는 문서곡괭이가 `파싱용공고문`에서 가져온 값 중 선택해서 보여준다.
이 값은 단독평가종목 규칙을 만들 때 필요하다.

예시:

```text
서버공고 `전문건설_주력분야=토공`
파싱용공고문에서도 `토공` 확인
  -> 단독평가종목 계산에서 주력분야 `토공`을 참고
```

## 종목묶기 상세 규칙

표시용 `종목`, 심사용 `단독평가종목`, `종목세부JSON`을 만드는 상세 규칙은 아래 문서를 따른다.

```text
docs/rules/종목묶기룰.md
```

예시:

```text
공고문 `금속창호지붕(주력분야: 금속구조물)`
  -> 종목=`금속창호지붕`
  -> 단독평가종목=`금속창호지붕_금속구조물`
```

## 설정 관리

종목 판단에 필요한 기준 파일은 웹의 설정 화면에서 추가, 변경, 수정할 수 있어야 한다.

예시:

```text
`data/jongmok.json` 또는 종목 매핑 파일을 코드에 고정하지 않고,
설정 화면에서 `전기공사업 등록업체 -> 전기` 매핑을 수정할 수 있어야 한다.
```

## 아직 필요한 설명

종목 묶기 규칙의 큰 구조는 `종목묶기룰.md`에 기록했다.
남은 미정사항은 `docs/09_미정사항.md`에서 예시와 함께 관리한다.

예시:

```text
`종목모두보유=1`일 때 `건축/토목`을 `(건축,토목)`으로 강제할지,
`공동도급형태=분담이행`일 때 괄호 묶기를 유지할지 같은 판단이 필요하다.
```
