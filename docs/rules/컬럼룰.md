# 컬럼 룰

## 목적

각 컬럼은 값이 어떻게 채워지는지 설명과 실행 규칙을 함께 가진다.

컬럼 룰은 아래 5개 필드를 중심으로 작동한다.

```text
처리방법
우선순위
참조방법
참조메모
선택목록
```

## 처리방법

`처리방법`은 누가 값을 만드는지 정한다.

| 값 | 의미 | 예시 |
|---|---|---|
| 수집 | 외부 API, 파서, LLM 등 수집 단계에서 받은 값을 사용 | `공고번호`, `발주처`, `추정가격`, `입찰방식` |
| 계산 | 다른 컬럼이나 마스터데이터에서 파생 | `발주처코드`, `원발주처`, `예상투찰금액` |
| 둘다 | 수집과 계산을 모두 시도한 뒤 우선순위로 선택 | `공동도급산정_시평액`처럼 수집값이 있으면 쓰고 없으면 계산 |
| 없음 | 아직 분류하지 않은 상태 | 새 컬럼 추가 직후 |

예시:

```text
`발주처코드`는 `발주처`를 마스터데이터에서 조회하므로 처리방법은 `계산`이다.
`공고번호`는 서버공고에서 받은 값을 그대로 쓰므로 처리방법은 `수집`이다.
```

## 우선순위

`우선순위`는 여러 출처가 같은 컬럼을 채울 때 누구를 믿을지 정한다.
우선순위는 컬럼마다 사용자가 지정한다.
최종 목표는 문서곡괭이 파서가 대부분의 값을 채우는 것이다.

| 값 | 의미 | 예시 |
|---|---|---|
| 문서곡괭이 | 파서가 공고문 HTML에서 추출한 값을 우선 사용 | `입찰방식`, `공사명`, `종목` |
| LLM | AI가 본문을 읽고 추출한 값을 우선 사용 | `A값` |
| 계산 | 계산 룰 결과를 우선 사용 | 처리방법이 `둘다`인 일부 컬럼 |
| 없음 | 출처 다툼이 없거나 아직 미정 | 계산 전용 컬럼, 신규 컬럼 |

예시:

```text
`A값`은 LLM 우선, 문서곡괭이 fallback으로 볼 수 있다.
`발주처코드`는 계산 전용이므로 우선순위가 없어도 된다.
```

주의:

```text
우선순위는 자동 후보값끼리의 선택 기준이다.
사람이 직접 입력하고 확정한 값은 사람입력값 층에 따로 보관하고, 필요한 컬럼만 재전처리한다.
```

## 참조방법

`참조방법`은 계산 룰의 본체다.
처리방법이 `계산` 또는 `둘다`일 때 활성화한다.

JSON 형식으로 기록하고, 룰 엔진이 이 JSON을 읽어서 값을 만든다.

| 유형 | 의미 | 예시 항목 |
|---|---|---|
| lookup | 마스터 테이블을 조회 | `발주처코드` |
| expr | 다른 컬럼 그대로 사용하거나 수식 계산 | `원발주처`, `예상투찰금액` |
| cond | 조건별로 다른 값을 선택 | `참여불가_신용평가_대표사` |
| finalize_1차 | 별도 후처리 hook | `적격평가기준_세부` |

예시:

```json
{"type":"lookup","table":"발주처코드","key_col":"발주처명","key_value":"발주처","return_col":"코드"}
```

위 룰은 `발주처` 값을 기준으로 `발주처코드` 마스터에서 코드를 찾아 채운다.

예시:

```json
{"type":"expr","need":["난이도계수"],"expr":"난이도계수 * 10"}
```

위 룰은 `난이도계수=1.5`이면 결과값을 `15`로 계산한다.

## 참조메모

`참조메모`는 사람을 위한 자연어 설명이다.
코드는 읽지 않고, 룰 디버깅과 인수인계를 위해 반드시 적는다.

| 항목 | 참조메모 예시 |
|---|---|
| 발주처코드 | 마스터데이터의 발주처코드 참조, 발주처명에서 코드 반환 |
| 원발주처 | 적격발주처와 같은 값 |
| 참여불가_신용평가_대표사 | PQ 입찰일 때 대표사 신용평가 제한값 |
| 일반_기타_전문 | 단독평가종목의 첫 종목을 종목 카탈로그에서 조회 |

예시:

```text
1년 뒤 다시 봐도 `왜 이 값이 들어갔는지` 알 수 있도록 처리 이유와 예시를 남긴다.
```

## 선택목록

`선택목록`은 드롭다운 옵션과 입력 검증 기준이다.
콤마 또는 줄바꿈으로 값을 기록한다.

선택목록이 있으면 아래 UI에 반영한다.

```text
공고 상세 모달 우측 패널
공고관리 메인표 셀 편집
사용자 입력값 검증
```

예시:

| 항목 | 선택목록 예시 |
|---|---|
| 입찰방식 | 적격심사, 수의계약, 종심제 |
| 참여형태 | 단독, 협정 |
| 공동도급형태 | 공동도급, 분담이행, 주계약자관리 |
| 지역제한 | 전국, 지역제한 |

검증 예시:

```text
`입찰방식` 선택목록이 `적격심사, 수의계약, 종심제`인데
사용자가 `종합심사`라고 직접 입력하면 확인 배지를 표시한다.
```

## 표시 제어

컬럼 수가 많으므로 화면 표시 여부는 체크박스로 동적으로 제어한다.

| 필드 | 의미 |
|---|---|
| 공고관리 표시 | 공고관리 메인표에 보여줄지 여부 |
| 상세정보입력 | 상세 모달 또는 입력 패널에 보여줄지 여부 |

예시:

```text
`공고번호`, `공사명`, `입찰일`은 공고관리 표에 표시한다.
`종목세부JSON`처럼 긴 값은 상세정보에서만 보이게 할 수 있다.
```

## 종목 기본 개념

`종목`, `단독평가종목`, `종목세부JSON`은 한몸으로 관리한다.
상세 규칙은 `docs/rules/종목룰.md`를 따른다.

## 주요 확정 규칙

| 항목 | 확정 규칙 | 예시 |
|---|---|---|
| 동일실적_평가여부 | 동일실적 컬럼에 평가비율이 있으면 `1`, 없으면 `0` | 동일실적 평가비율이 30%이면 `1` |
| 적격발주처 | 서버공고에 컬럼으로 올 수 있지만 값이 항상 채워진다고 보지 않는다. 값이 있으면 홈페이지 연결 기준 후보로 사용 | `적격발주처=조달청`이면 나라장터 연결 후보 |
| 원발주처 | 실제 평가에 사용할 적격심사기준 발주처 | `한국전력공사(배전)`, `조달청` |
| 적격평가기준_세부 | A4 적격심사세부기준 API 호출 기준 | `행자부_별표-4`, `한국가스공사_별지-7` |
| AI컬럼 | AI가 채운 컬럼 목록을 기록하는 로그 | `A값,상호진출_일반전문` |
| 공고확인 | 사용자에게 확인 키워드를 보여주는 태그 | `현장확인`, `기초금액발표전` |
| 공고확인_내용 | 사용자가 입력하거나 파싱한 설명 텍스트 | `현장실사 후 계약상대자 결정` |
| 특수실적 계열 | 공고확인 계열과 같이 파싱 가능하면 자동 입력, 어려우면 사람이 입력 | `최근 10년 이내 준공실적...` |
| 단독평가종목 | 종목을 평가 가능하게 분해했다가 재조립한 값. 상세 규칙은 종목룰 문서 참조 | `(토목,산업환경설비)` |
| 종목세부JSON | 단독평가종목을 종목별 객체로 풀어낸 값. 단독평가종목과 한몸 | 종목별 비율, 실적기간, 만점실적 |
| 전문건설_주력분야 | 서버공고 수집값 또는 파싱용공고문 파싱값 중 선택 | `토공`, `포장` |
| 앞_공고번호 | 공고번호에서 차수 앞부분 | `R26BK01537599-000` -> `R26BK01537599` |
| 공고번호_차수 | 공고번호 뒤 차수 숫자 | `R26BK01537599-000` -> `0` |
| 재입찰번호 | 재입찰 구분 번호 | 기본값 `0` |

## 단독평가종목과 종목세부JSON

`단독평가종목`과 `종목세부JSON`은 한 묶음으로 본다.
상세 규칙은 `docs/rules/종목룰.md`에 계속 추가한다.

```text
종목 원본
  -> 평가 가능하도록 분해
  -> 단독평가종목 생성
  -> 종목세부JSON에 종목별 세부 정보로 전개
```

예시:

```text
`단독평가종목=(토목,산업환경설비)`이면
`종목세부JSON`에는 토목과 산업환경설비가 각각 별도 객체로 들어간다.
```

## 설정 마스터

전처리 기준이 되는 파일은 웹 설정 화면에서 추가, 변경, 수정할 수 있어야 한다.
상세 기준은 `docs/rules/설정마스터룰.md`를 따른다.

예시:

```text
적격발주처 파일, 공고확인 선택목록, 특수실적 선택목록, 종목 매핑 파일은
코드에 고정하지 않고 설정에서 관리한다.
```

## 적격심사기준 흐름

`적격발주처`, `원발주처`, `적격평가기준_세부`의 상세 의미와 처리 순서는
`docs/rules/적격심사기준룰.md`를 따른다.

예시:

```text
서버공고 또는 전처리 결과 `적격발주처=한국전력공사`인 공고라도
실제 심사기준이 조달청이면 `원발주처=조달청`으로 다시 잡고
그 다음 `적격평가기준_세부`를 찾는다.
```

## 검증용 컬럼

아래 컬럼은 과거 수집 검증에 사용했던 컬럼이다.
검증 체계를 다시 설계할 수 있으므로 지금은 규칙을 확정하지 않는다.

```text
금액정보
_v1_검증_상태
_v1_검증_메시지
_v1_적격_1차_매칭
_v1_적격_2차_매칭
_v1_적격발주처_상태
_v1_적격발주처_사유
문서곡괭이컬럼
```

예시:

```text
`_v1_검증_상태`는 나중에 새 검증 흐름을 정할 때 다시 상태값을 설계한다.
```
