# 페이지 구성

## 기본 메뉴

```text
수집
전처리
사람입력
최종출력
룰관리
```

## 설계 원칙

발주처, 업무유형, 종목, 종목그룹은 페이지가 아니라 필터와 룰로 처리한다.

나쁜 예시:

```text
발주처A 시설 페이지
발주처A 용역 페이지
발주처B 물품 페이지
```

좋은 예시:

```text
수집 페이지
- 발주처: 서버공고의 발주처 컬럼
- 업무유형: 시설
- 종목그룹: 전기/통신/소방
- 상태: 검토필요
```

## 페이지별 역할

| 페이지 | 역할 | 예시 |
|---|---|---|
| 수집 | 원본 수집 상태 확인 | 수집 성공, 실패, 첨부파일 다운로드 여부 |
| 전처리 | 표준 컬럼 변환 결과 확인 | 원본 `bidNtceNm`이 표준 `공고명`으로 매핑됨 |
| 사람입력 | 빈값과 충돌값을 사람이 검증 | `업종=기타`를 `전기공사업`으로 수정 |
| 최종출력 | 검증 완료 데이터를 엑셀로 출력 | 검토완료만 출력 |
| 룰관리 | 매핑, 선택, 재전처리, 출력 룰 관리 | `공고명` 변경 시 재처리 대상 지정 |

## 화면 복잡도 제한

하나의 작업에도 탭이 많이 생길 수 있으므로, 메인 페이지를 늘리지 않고 화면 내부 패널로 처리한다.

사람입력 화면 예시:

```text
왼쪽: 공고 목록과 필터
가운데: 표준 컬럼, 후보값, 사람 입력값
오른쪽: 공고문, 근거, 적용 룰
하단: 재전처리 결과와 처리 로그
```
