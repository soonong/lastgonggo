## 구현 내용

처음 정한 흐름인 `수집 -> 전처리 -> 사람입력 -> 최종출력`을 메인 화면에서 바로 볼 수 있도록 `작업 흐름` 탭을 추가했다.

흐름 정의는 코드에 고정하지 않고 `data/settings/작업흐름.csv`에서 읽는다.

화면 구성:

1. 전체 흐름 요약
2. 수집/전처리/사람입력/최종출력 row 수
3. 룰관리 설정 row 수
4. 전처리 세부 구현 단위 카드
5. 각 카드에서 연결 화면으로 이동하는 버튼
6. 마지막 곡괭이질 상태와 검증 상태

## 작업흐름 데이터

`data/settings/작업흐름.csv`는 최신 작업흐름 기준으로 아래 구현 단위를 가진다.

- 서버공고 수집
- 곡괭이전나라시
- A3 공고문 정규화
- 문서곡괭이
- 공고체크 조건값 채우기
- 값 선택 / 컬럼 우선순위
- 종목묶기
- 곡괭이후나라시 / 계산맨
- 적격심사 1차 매칭
- 적격심사 2차 매칭
- A4 적격심사세부기준 조회
- 검증맨
- 사람입력 / 재전처리
- 최종출력
- 룰관리

## 2026-05-28 갱신

- 작업흐름 탭을 13개 구현 단위에서 15개 구현 단위로 갱신했다.
- 문서곡괭이 전에 `A3 공고문 정규화` 단계를 분리했다.
- 2차 매칭 전에 필요한 `공고체크 조건값 채우기` 단계를 분리했다.
- 수집 설명은 `서버공고 API 수집`으로 명확히 하고, A3/A4는 수집 원본이 아니라 전처리 참조 API라고 표시했다.

## 2026-05-28 수집/전처리 탭 분리

- 우측 `AI 로그` 패널은 접기/펼치기 버튼으로 공간을 줄일 수 있게 했다.
- `입찰공고 원본` 탭에는 운영 버튼인 `API 호출`, `곡괭이전나라시`, `모두 삭제`만 둔다. 샘플 로드 기능과 `local-sample` 표시는 제거했다.
- `API 호출`은 A1 결과 중 `자동수집여부=1`인 row만 적재한다. 입력일 필터가 있으면 해당 날짜 포함 이후 row만, 종목 필터가 있으면 해당 종목 포함 row만 적재한다.
- `곡괭이전나라시`는 서버공고 raw를 나라시해서 `입찰공고 전처리` 탭으로 넘긴다.
- `입찰공고 전처리` 탭에는 `문서곡괭이질 시작`만 둔다. 이 버튼이 A3 문서곡괭이와 후속 나라시/검증을 실행하고 `공고관리` 대상 row를 만든다.
- `공고관리` 탭은 전처리 row를 종목별 건수로 묶어 보여주고, `곡괭이후나라시` 버튼으로 `2차분류 완료` 탭으로 이관한다.
- 이관된 종목 그룹에는 `이관완료` 표시를 붙인다.

## 2026-05-28 공고수집 화면 압축

- `moduleKey 연결됨`, `local-sample`, 별도 진행결과 박스는 제거하고 표 제목줄 한 줄에 진행상태와 통계를 모았다.
- 표 제목줄은 `서버공고 전체 컬럼 · API 호출 완료 639/3267 · 전체 639건 · 검색 639건 · 화면 표시 200건 · 컬럼 118 · 빈칸 21,214` 같은 형식으로 표시한다.
- `API 호출`, `곡괭이전나라시`, `문서곡괭이질`, `곡괭이후나라시` 진행 중에는 같은 제목줄에 `현재작업 done/total` 형식으로 표시한다.
- `공고수집 > 공고분류컬럼 설정`의 `id`를 다시 표시하고 수정 가능하게 했다. 메인 4개 탭의 컬럼 순서는 이 `id` 오름차순을 따른다.
- A1 수집 직후 `공고분류컬럼 설정`의 `항목/표시형식`과 실제 서버공고 값을 비교해 표시형식 불일치가 있으면 제목줄과 로그에 요약한다.
