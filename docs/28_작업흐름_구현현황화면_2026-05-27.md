## 구현 내용

처음 정한 흐름인 `수집 -> 전처리 -> 사람입력 -> 최종출력`을 메인 화면에서 바로 볼 수 있도록 `작업 흐름` 탭을 추가했다.

흐름 정의는 코드에 고정하지 않고 `data/settings/작업흐름.csv`에서 읽는다.

화면 구성:

1. 전체 흐름 요약
2. 수집/전처리/사람입력/최종출력 row 수
3. 룰관리 설정 row 수
4. 전처리 세부 구현 단위 카드
5. 각 카드에서 연결 화면으로 이동하는 버튼
6. 마지막 곡괭이질 상태와 검증 상태

## 작업흐름 데이터

`data/settings/작업흐름.csv`는 아래 구현 단위를 가진다.

- 서버공고 수집
- 곡괭이전나라시
- 문서곡괭이
- 값 선택 / 컬럼 우선순위
- 종목묶기
- 곡괭이후나라시 / 계산맨
- 적격심사 1차 매칭
- 적격심사 2차 매칭
- A4 적격심사세부기준 조회
- 검증맨
- 사람입력 / 재전처리
- 최종출력
- 룰관리
