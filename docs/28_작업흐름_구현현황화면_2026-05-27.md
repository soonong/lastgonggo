## 구현 내용

처음 정한 흐름인 `수집 -> 전처리 -> 사람입력 -> 최종출력`을 메인 화면에서 바로 볼 수 있도록 `작업 흐름` 탭을 추가했다.

흐름 정의는 코드에 고정하지 않고 `data/settings/작업흐름.csv`에서 읽는다.

화면 구성:

1. 전체 흐름 요약
2. 수집/전처리/사람입력/최종출력 row 수
3. 룰관리 설정 row 수
4. 전처리 세부 구현 단위 카드
5. 각 카드에서 연결 화면으로 이동하는 버튼
6. 마지막 곡괭이질 상태와 검증 상태

- 서버공고 수집
- 곡괭이전나라시
- A3 공고문 정규화
- 문서곡괭이
- 공고체크 조건값 채우기
- 값 선택 / 컬럼 우선순위
- 종목묶기
- 곡괭이후나라시 / 계산맨
- 적격심사 1차 매칭
- 적격심사 2차 매칭
- A4 적격심사세부기준 조회
- 검증맨
- 사람입력 / 재전처리
- 최종출력
- 룰관리

## 2026-05-28 갱신

- 작업흐름 탭을 13개 구현 단위에서 15개 구현 단위로 갱신했다.
- 문서곡괭이 전에 `A3 공고문 정규화` 단계를 분리했다.
- 2차 매칭 전에 필요한 `공고체크 조건값 채우기` 단계를 분리했다.
- 수집 설명은 `서버공고 API 수집`으로 명확히 하고, A3/A4는 수집 원본이 아니라 전처리 참조 API라고 표시했다.

## 2026-05-28 수집/전처리 탭 분리

- 우측 `AI 로그` 패널은 접기/펼치기 버튼으로 공간을 줄일 수 있게 했다.
- `입찰공고 원본` 탭에는 운영 버튼인 `API 호출`, `곡괭이전나라시`, `모두 삭제`만 둔다. 샘플 로드 기능과 `local-sample` 표시는 제거했다.
- `API 호출`은 A1 결과 중 `자동수집여부=1`인 row만 적재한다. 입력일 필터가 있으면 해당 날짜 포함 이후 row만, 종목 필터가 있으면 해당 종목 포함 row만 적재한다.
- `곡괭이전나라시`는 서버공고 raw를 나라시해서 `입찰공고 전처리` 탭으로 넘긴다.
- `입찰공고 전처리` 탭에는 `문서곡괭이질 시작`만 둔다. 이 버튼이 A3 문서곡괭이와 후속 나라시/검증을 실행하고 `공고관리` 대상 row를 만든다.
- `공고관리` 탭은 전처리 row를 종목별 건수로 묶어 보여주고, `곡괭이후나라시` 버튼으로 `2차분류 완료` 탭으로 이관한다.
- 이관된 종목 그룹에는 `이관완료` 표시를 붙인다.

## 2026-05-28 공고관리 종목 검수 큐

- 공고관리의 종목 묶음은 복잡한 카탈로그 매칭이 아니라 `종목` 컬럼을 `/`, `,`, 괄호 기준으로 나눈 토큰으로 만든다.
- 예: `종목=(건축,토목)/토건`이면 `건축`, `토목`, `토건` 세 묶음에 각각 포함된다.
- 사람이 한 종목 묶음을 검수하고 `2차분류 완료로 이동`을 누르면 해당 묶음의 미완료 공고만 곡괭이후나라시 후 `2차분류 완료`로 이동한다.
- 한 번 2차분류 완료로 이동한 공고번호는 다른 종목 묶음에도 포함되어 있더라도 공고관리 대상에서 제외한다.
- 종목 묶음에는 남은 건수와 이미 이관된 건수를 함께 표시하고, 남은 건수가 0이면 `이관완료`로 표시한다.
- 종목 묶음 선택은 별도 칩 영역을 만들지 않고 상단 `종목` 목록상자에서 한다. 공고관리 탭에서만 목록상자 옵션이 `종목명 n건 · n건 이관` 형식으로 바뀐다.

| 입찰공고 전처리 | 곡괭이전나라시가 끝난 row를 확인한다. 여기서 문서곡괭이질을 실행해 A3 본문 파싱과 후속 전처리를 한다. | `문서곡괭이질 시작`, 진행 중 `중단` | 입력일, 종목, 검색 | 문서곡괭이 결과와 검증 상태가 반영된 row를 `공고관리` 대상으로 만든다. | A1 API 호출, 사람 검수 완료 이관 |

### 탭 간 데이터 이동 원칙

```text
입찰공고 원본
  API 호출
  -> 곡괭이전나라시
입찰공고 전처리
  -> 문서곡괭이질 시작
공고관리
  -> 종목별 사람 검수
  -> 2차분류 완료로 이동
2차분류 완료
  -> 엑셀 다운로드
```

- 원본 탭은 raw 수집과 곡괭이전나라시까지만 한다.
- 전처리 탭은 문서곡괭이질과 전처리 결과 확인까지만 한다.
- 공고관리 탭은 사람 검수와 선택 종목 이관까지만 한다.
- 2차분류 완료 탭은 완료 데이터 확인과 출력만 한다.
- 룰관리는 모든 단계의 기준표이며, 처리 단계 자체가 아니다.

## 2026-05-29 공고관리 상세모달 1차 연결

공고관리의 작업 흐름은 `종목 선택 -> 공사명 더블클릭 -> 상세모달 입력 -> 저장 -> 2차분류 완료로 이동`을 기준으로 둔다.

- 공고관리 탭의 상단 `종목` 목록상자는 사람 검수 큐 선택기로 사용한다.
- 공고관리 테이블에서는 `공사명` 셀을 더블클릭하면 상세모달을 연다. 다른 탭은 기존처럼 row 클릭으로 상세 확인을 유지한다.
- 상세모달 좌측은 공고문 영역이다. 기본 탭은 `공고문`이고, 우측 `첨부` 탭에서 파일을 누르면 좌측에 파일 탭이 추가된다.
- 상세모달 좌측 검색은 현재 공고문 텍스트를 대상으로 하이라이트한다.
- 상세모달 우측은 `추가`, `종목`, `기본`, `기타`, `첨부`, `적격심사기준` 탭으로 나뉜다.
- 우측 필드는 읽기 전용이 아니라 사람이 직접 수정할 수 있는 입력값이다.
- `저장 / 닫기`를 누르면 현재 단계의 같은 `공고번호` row에 수정값을 반영한다.

## 2026-05-29 테스트 CSV 가져오기/내보내기

문서곡괭이질은 시간이 오래 걸리므로 전처리/공고관리 단계별 테스트 데이터를 CSV로 저장하고 다시 불러올 수 있게 한다.

- `입찰공고 전처리` 탭에는 `테스트 내보내기`, `테스트 가져오기` 버튼을 둔다.

- 공고관리 탭에서 가져온 CSV는 `humanRows`를 교체하고 공고관리 탭에서 바로 검수/상세모달 테스트를 이어간다.
- 내보내기는 브라우저 다운로드와 서버 export 저장을 같이 시도한다. 서버 저장이 실패해도 브라우저 다운로드는 유지한다.
