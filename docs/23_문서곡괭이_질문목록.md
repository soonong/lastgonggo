# 문서곡괭이 상세 질문 목록

작성일: 2026-05-24

## 목적

문서곡괭이는 파싱용공고문에서 값을 자동 추출하는 핵심 엔진이다.
기존 문서곡괭이 참고자료는 흐름과 매처 개념만 참고하고, 새 구조는 질문을 통해 다시 확정한다.

답변 방식:

```text
질문 번호 기준으로 답변하면 된다.
예: PM-01 = A3 먼저 보고 실패하면 A2 첨부 공고문을 본다.
```

모든 질문은 예시를 기준으로 판단한다.

---

# 1. 본문 입력 범위

## PM-01. 문서곡괭이의 1순위 본문은 A3 파싱용공고문인가?

예시:

```text
A3 파싱용공고문에는 공고 본문 HTML이 있음.
A2 첨부파일에는 공고문 HWP/PDF가 있음.

기본은 A3만 먼저 보고,
A3에 값이 없을 때 A2 첨부파일을 fallback으로 볼까?
```

## PM-02. A5 웹용공고문은 문서곡괭이 입력으로 쓰나?

예시:

```text
A3 파싱용공고문에는 참가자격 문단이 누락됨.
A5 웹용공고문에는 화면용 상세 HTML에 참가자격이 있음.

A5는 사람이 보는 용도만으로 둘까?
아니면 A3가 불완전할 때 문서곡괭이 fallback으로 쓸까?
```

## PM-03. 첨부파일이 여러 개면 우선순위는?

예시:

```text
A2 첨부파일 목록:
1) 입찰공고문.hwp
2) 과업지시서.pdf
3) 내역서.xlsx
4) 시방서.hwp

문서곡괭이는 `입찰공고문` 이름이 들어간 파일을 먼저 보고,
그 다음 `공고`, `입찰`, `안내` 파일을 볼까?
```

## PM-04. 첨부파일 중 파싱 대상에서 제외할 파일명 패턴은?

예시:

```text
파일명=내역서.xlsx
파일명=설계도면.pdf
파일명=청렴계약이행각서.hwp

이런 파일은 참가자격/특수실적 추출에 방해가 되니
기본 파싱 대상에서 제외할까?
```

## PM-05. 여러 본문에서 같은 컬럼을 찾으면 어떤 근거를 우선할까?

예시:

```text
A3 본문:
추정가격=1억

A2 공고문 첨부:
추정가격=1억 1천만

둘이 다르면 A3를 우선할까?
첨부 공고문을 우선할까?
둘 다 후보로 남기고 검토필요로 둘까?
```

## PM-09. 줄바꿈과 표 셀은 문단으로 어떻게 나눌까?


## PM-10. 원문 근거는 정규화 전 원문으로 보여줄까?


## PM-16. 정규식 입력은 허용할까?

```text

일반 운영자 화면에서는 정규식을 막고,
```

```text

```

# 5. 날짜·금액·비율 추출

# 18. 확정 반영: 조건판단형태 범위 정의

조건판단형태 가이드는 각 타입별로 아래 범위를 함께 관리한다.

| 항목 | 확정 내용 | 예시 |
|---|---|---|
| 정규화범위 | 문서곡괭이가 우선 검색할 단위 | 원문줄, 문단, 섹션, 표 |
| 키워드매칭방식 | 조건판단형태별 키워드 인정 방식. 하나만 선택 | 키워드일치, 키워드포함, 와일드카드, 별칭, 헤더라벨, 마스터검색 |
| 우선섹션분류 | 섹션 분류가 있으면 먼저 좁혀서 검색 | 금액정보 섹션에서 추정가격 검색 |
| 제외검사범위 | 제외키워드를 비교할 범위 | 같은 문단, 매칭 표, 매칭 섹션 |
| 근거저장단위 | 검수자가 확인할 원문 근거 단위 | 매칭 문단, 매칭 행, 매칭 섹션 |

예시:

```text
조건판단형태=3_1
검색키워드=상호진출
키워드매칭방식=키워드포함
제외키워드=허용하지 않, 불가, 미허용
제외검사범위=같은 문단

본문: 상호시장 진출은 허용하지 않음
결과: 제외키워드 통과 실패 -> 빈값
```

이 구조는 명확한 키워드는 빠르게 잡고, 유사 키워드나 부정 표현은 제외키워드와 범위로 보수적으로 거르기 위한 기준이다.
